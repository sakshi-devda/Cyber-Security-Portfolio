# Threats and Vulnerabilities

## 📌 Definition
A **Threat** is any potential danger that can exploit a system weakness.  
A **Vulnerability** is a weakness or flaw in a system that can be exploited by threats.

---

## 🧨 Threats
Threats are actions or events that can cause harm.

### Examples:
- Hackers
- Malware
- Phishing attacks
- Natural disasters

---

## ⚠️ Vulnerabilities
Vulnerabilities are weaknesses in a system.

### Examples:
- Weak passwords
- Unpatched software
- Misconfigured systems
- Outdated OS

---

## 🔗 Relationship
Threat + Vulnerability = Attack

If a system has a weakness, threats can exploit it.

---

## 🧠 Real World Example
- Vulnerability: Weak WiFi password
- Threat: Hacker nearby
- Result: Unauthorized access to network

---

## 🛡️ Prevention
- Regular software updates
- Strong passwords
- Security patches
- System hardening
- Monitoring systems

---

## 🎯 Key Takeaway
Without vulnerabilities, threats cannot succeed.
