# SIEM (Security Information and Event Management)

## 📌 Definition
SIEM is a security solution that collects, analyzes, and correlates security data from multiple sources in real time to detect threats and generate alerts.

---

## 🎯 Purpose of SIEM
- Centralized security monitoring
- Detect suspicious activities
- Generate alerts for incidents
- Support incident response and forensics

---

## ⚙️ How SIEM Works
1. Collects logs from different sources
   - Servers
   - Firewalls
   - Applications
   - Network devices

2. Normalizes data into a standard format

3. Correlates events to identify patterns

4. Generates alerts for suspicious behavior

---

## 🧠 Real World Example
- Multiple failed login attempts detected from same IP
- SIEM flags it as brute force attack
- Security team gets alert

---

## 🧱 Key Components of SIEM

### 1. Log Collection
Gathers data from various systems.

### 2. Event Correlation
Connects related events to detect attacks.

### 3. Alerting
Notifies security teams about threats.

### 4. Reporting
Provides dashboards and security reports.

---

## 🛡️ Benefits of SIEM
- Real-time threat detection
- Faster incident response
- Centralized visibility
- Compliance reporting

---

## ⚠️ Limitations
- Expensive to implement
- Requires skilled analysts
- Can generate false positives

---

## 🧠 Popular SIEM Tools
- Splunk
- IBM QRadar
- Microsoft Sentinel
- ArcSight

---

## 🎯 Key Takeaway
SIEM acts as the central brain of a security monitoring system by collecting and analyzing logs from multiple sources.
