# Incident Response (IR) in Cyber Security

## 📌 Definition
Incident Response is a structured process used to detect, respond to, and recover from cyber security incidents.

A security incident is any event that compromises confidentiality, integrity, or availability of systems or data.

---

## 🎯 Objectives of Incident Response
- Minimize damage from attacks
- Reduce recovery time and cost
- Prevent future incidents
- Ensure business continuity

---

## ⚙️ Phases of Incident Response

### 1. Preparation
Setting up tools, policies, and training before any incident occurs.

**Example:**
- Installing SIEM tools
- Creating response team (CSIRT)

---

### 2. Identification
Detecting and confirming a security incident.

**Example:**
- SIEM alert for unusual login activity

---

### 3. Containment
Limiting the spread of the attack.

**Example:**
- Disconnecting infected system from network

---

### 4. Eradication
Removing the root cause of the incident.

**Example:**
- Deleting malware
- Patching vulnerability

---

### 5. Recovery
Restoring systems back to normal operation.

**Example:**
- Restoring data from backup
- Bringing servers back online

---

### 6. Lessons Learned
Analyzing the incident to improve future security.

**Example:**
- Updating security policies
- Improving monitoring rules

---

## 🧠 Real World Example
A company detects ransomware attack:
- SIEM alerts unusual file encryption
- System is isolated
- Malware removed
- Data restored from backup
- Security rules improved

---

## 🛡️ Importance of Incident Response
- Reduces impact of cyber attacks
- Improves organizational resilience
- Helps in legal and compliance requirements

---

## 🎯 Key Takeaway
Incident Response is a structured lifecycle that helps organizations handle cyber attacks efficiently and recover quickly.
