# Linux Networking Commands

## 📌 What are Linux Networking Commands?

Linux networking commands are used to view, configure, test, and troubleshoot network connections. These commands help administrators and cybersecurity professionals monitor network activity and diagnose connectivity issues.

---

## 🎯 Why are Networking Commands Important?

- Check network connectivity.
- View IP addresses and network interfaces.
- Troubleshoot network problems.
- Verify DNS resolution.
- Test communication between systems.
- Essential for cybersecurity investigations.

---

## 🌐 Common Linux Networking Commands

| Command | Purpose |
|---------|---------|
| ip addr | Display IP addresses and network interfaces |
| ip route | Display routing table |
| ping | Test network connectivity |
| hostname | Display system hostname |
| hostname -I | Display system IP address |
| ifconfig | Display network interface information (legacy command) |
| netstat | Display network connections (legacy command) |
| ss | Display active network connections |
| traceroute | Trace the path to a destination |
| nslookup | Query DNS records |
| dig | Perform detailed DNS lookups |

---

## 📖 Important Commands Explained

### 1. ip addr

Displays IP addresses and network interface details.

Example:

```bash
ip addr
```

---

### 2. ping

Tests connectivity between your system and another host.

Example:

```bash
ping google.com
```

---

### 3. hostname

Displays the system hostname.

Example:

```bash
hostname
```

---

### 4. hostname -I

Displays the IP address assigned to the system.

Example:

```bash
hostname -I
```

---

### 5. ip route

Displays the routing table.

Example:

```bash
ip route
```

---

### 6. ss

Displays active TCP and UDP connections.

Example:

```bash
ss -tuln
```

---

### 7. traceroute

Shows the path packets take to reach a destination.

Example:

```bash
traceroute google.com
```

---

### 8. nslookup

Queries DNS information.

Example:

```bash
nslookup google.com
```

---

### 9. dig

Provides detailed DNS information.

Example:

```bash
dig google.com
```

---

## 🛡️ Cybersecurity Perspective

Networking commands help security professionals:

- Verify network connectivity.
- Detect suspicious connections.
- Identify open listening ports.
- Troubleshoot DNS issues.
- Analyze routing problems during incident response.

---

## ⭐ Best Practices

- Use `ip` commands instead of older `ifconfig` whenever possible.
- Verify DNS before assuming a website is unreachable.
- Monitor active connections regularly.
- Investigate unexpected listening ports.

---

## 💼 Real-World Example

A SOC Analyst receives an alert that a server cannot reach an internal application.

They may use:

- `ping`
- `ip addr`
- `ip route`
- `nslookup`
- `ss`

to identify whether the issue is related to connectivity, routing, DNS, or network services.
