# Linux Users and Groups

## 📌 What are Users and Groups?

Linux is a multi-user operating system where multiple users can access the same system. To manage access and security, Linux uses users and groups.

A **user** represents an individual account, while a **group** is a collection of users who share common permissions.

---

## 🎯 Why are Users and Groups Important?

- Control access to files and directories.
- Improve system security.
- Simplify permission management.
- Support multi-user environments.
- Prevent unauthorized access.

---

## 👤 Types of Users

### 1. Root User

The root user is the administrator of the Linux system.

Features:
- Has complete control over the system.
- Can access all files and directories.
- Can install or remove software.
- Can create or delete users.

Example:

```bash
root
```

---

### 2. Regular User

A regular user has limited permissions.

Features:
- Can access personal files.
- Cannot modify critical system files.
- Requires administrative privileges for system changes.

Example:

```bash
kali
```

---

### 3. System User

System users are created automatically for running services and applications.

Examples:

- daemon
- www-data
- nobody

---

## 👥 What is a Group?

A group is a collection of users.

Instead of assigning permissions individually, permissions can be assigned to a group.

Example:

Developers

- User1
- User2
- User3

All members inherit the group's permissions.

---

## 📂 User Information File

Linux stores user account information in:

```text
/etc/passwd
```

This file contains:

- Username
- User ID (UID)
- Group ID (GID)
- Home directory
- Default shell

---

## 📂 Group Information File

Linux stores group information in:

```text
/etc/group
```

This file contains:

- Group name
- Group ID
- Group members

---

## 🔑 User ID (UID)

Every user has a unique User ID.

Examples:

- Root User → UID 0
- Regular Users → UID 1000 and above (commonly)

---

## 🔑 Group ID (GID)

Every group has a unique Group ID used to identify it.

---

## 🛡️ Cybersecurity Perspective

Managing users and groups properly reduces unauthorized access.

Security analysts often:

- Review user accounts.
- Check inactive users.
- Remove unnecessary privileges.
- Verify group memberships.
- Audit administrative accounts.

---

## ⭐ Best Practices

- Follow the Principle of Least Privilege.
- Avoid logging in as the root user for daily tasks.
- Remove unused accounts.
- Assign users to appropriate groups.
- Regularly audit user permissions.

---

## 💼 Real-World Example

A company has separate groups for:

- Developers
- HR
- Finance
- Security Team

Each group receives only the permissions required for its work, reducing the risk of unauthorized access.
