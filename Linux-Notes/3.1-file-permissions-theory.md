# Linux File Permissions

## 📌 What are File Permissions?

File permissions in Linux define who can read, write, or execute a file or directory. They are an essential security feature that protects files from unauthorized access or modification.

---

## 🎯 Why are File Permissions Important?

- Protect sensitive files from unauthorized users.
- Control who can view, modify, or execute files.
- Improve system security.
- Prevent accidental deletion or modification of important files.
- Essential for Linux administration and cybersecurity.

---

## 👥 Permission Categories

Linux permissions are assigned to three categories of users:

### 1. User (u)

The owner of the file.

### 2. Group (g)

Users who belong to the same group.

### 3. Others (o)

All other users on the system.

---

## 🔐 Permission Types

There are three basic permissions in Linux.

| Permission | Symbol | Value | Description |
|------------|--------|-------|-------------|
| Read | r | 4 | View the contents of a file. |
| Write | w | 2 | Modify or edit a file. |
| Execute | x | 1 | Run a file as a program or access a directory. |

---

## 📋 Understanding Permission Output

Example:

```bash
-rwxr-xr--
```

Breakdown:

```
- rwx r-x r--
│ │   │   │
│ │   │   └── Others
│ │   └────── Group
│ └────────── User (Owner)
└──────────── File Type
```

Meaning:

- User: Read, Write, Execute
- Group: Read, Execute
- Others: Read only

## 🔢 Numeric (Octal) Permissions

Linux also represents permissions using numbers.

| Permission | Value |
|------------|------:|
| Read | 4 |
| Write | 2 |
| Execute | 1 |

Examples:

| Number | Permission |
|--------|------------|
| 7 | rwx |
| 6 | rw- |
| 5 | r-x |
| 4 | r-- |

Example:

```text
755 = rwxr-xr-x
644 = rw-r--r--
700 = rwx------
```

---

## 🛡️ Cybersecurity Perspective

Incorrect file permissions can expose confidential data or allow attackers to modify important system files.

For example:

- A file with **777** permissions can be read, written, and executed by anyone, making it a major security risk.
- Security analysts regularly audit file permissions to identify misconfigurations.

---

## ⭐ Best Practices

- Follow the **Principle of Least Privilege**.
- Avoid giving **777** permissions unless absolutely necessary.
- Give users only the permissions they need.
- Regularly review file permissions.
- Protect sensitive configuration files.

---

## 💼 Real-World Example

A web server stores its configuration in configuration files.

If these files have incorrect permissions, an attacker might modify them to gain unauthorized access or execute malicious code.
