# Linux File Permissions Practical

## 🎯 Objective

Learn how to view, modify, and manage file permissions and ownership in Linux.

---

## 🧪 Lab Environment

- Operating System: Kali Linux
- Virtual Machine: VirtualBox
- Terminal: Bash

---

# 🖥️ Practical 1: View File Permissions

## Command

```bash
ls -l
```

## Purpose

Displays detailed information about files, including permissions.

## Screenshot

> ![ls -l](screenshots/ls-l-command.png)

## Explanation

The `ls -l` command shows file permissions, owner, group, file size, and last modification date.

---

# 🖥️ Practical 2: Change Permissions Using Symbolic Mode

## Command

```bash
chmod u+x filename
```

## Purpose

Adds execute permission for the file owner.

## Screenshot

> 📸 Screenshot will be added later.

## Explanation

`chmod` changes file permissions.  
`u+x` means **add execute permission to the user (owner).**

---

# 🖥️ Practical 3: Remove Write Permission

## Command

```bash
chmod g-w filename
```

## Purpose

Removes write permission from the group.

## Screenshot

> 📸 Screenshot will be added later.

## Explanation

`g-w` removes write permission from users belonging to the file's group.

---

# 🖥️ Practical 4: Change Permissions Using Numeric Mode

## Command

```bash
chmod 755 filename
```

## Purpose

Assigns permissions using numeric values.

## Screenshot

> 📸 Screenshot will be added later.

## Explanation

`755` means:

- Owner → Read, Write, Execute
- Group → Read, Execute
- Others → Read, Execute

---

# 🖥️ Practical 5: Set Read and Write Only

## Command

```bash
chmod 644 filename
```

## Purpose

Assigns secure permissions for most files.

## Screenshot

> 📸 Screenshot will be added later.

## Explanation

`644` means:

- Owner → Read, Write
- Group → Read
- Others → Read

---

# 🖥️ Practical 6: Change File Owner

## Command

```bash
sudo chown username filename
```

## Purpose

Changes the owner of a file.

## Screenshot

> 📸 Screenshot will be added later.

## Explanation

`chown` changes file ownership. Administrator privileges are usually required.

---

# 🖥️ Practical 7: Change File Group

## Command

```bash
sudo chgrp groupname filename
```

## Purpose

Changes the group ownership of a file.

## Screenshot

> 📸 Screenshot will be added later.

## Explanation

`chgrp` assigns a different group to the file.

# 🏋️ Practice Tasks

- View permissions of files in your current directory.
- Create a test file and change its permissions to `755`.
- Change the permissions to `644`.
- Add execute permission using symbolic mode.
- Remove write permission from the group.
- View the updated permissions using `ls -l`.

# 🎯 Key Takeaway

Linux file permissions are essential for securing files and directories. Learning how to use `chmod`, `chown`, and `chgrp` is a fundamental skill for Linux administration, ethical hacking, and cybersecurity professionals.
