# Linux Process Management

## 📌 What is a Process?

A process is a running instance of a program. Whenever you execute a command or start an application, Linux creates a process to perform that task.

For example:

- Opening Firefox creates a process.
- Running the `ping` command creates a process.
- Starting Apache or SSH creates one or more processes.

---

## 🎯 Why is Process Management Important?

- Monitor running programs.
- Identify resource-intensive processes.
- Stop unresponsive applications.
- Troubleshoot system issues.
- Detect suspicious or malicious processes.
- Essential for Linux administration and cybersecurity.

---

## 🧠 Process Lifecycle

A process generally goes through these stages:

1. Created
2. Running
3. Waiting
4. Stopped
5. Terminated

---

## 🆔 Process ID (PID)

Every running process has a unique Process ID (PID).

Example:

```
PID 1024 → Firefox
PID 2156 → SSH
```

Linux uses the PID to identify and manage processes.

---

## 📋 Common Process Management Commands

| Command | Purpose |
|----------|---------|
| ps | Display running processes |
| ps -ef | Display all running processes |
| top | Monitor processes in real time |
| htop | Interactive process viewer |
| kill | Terminate a process using PID |
| killall | Terminate processes by name |
| jobs | Display background jobs |
| bg | Resume a stopped job in the background |
| fg | Bring a background job to the foreground |

---

## 🔍 Understanding Process States

Common process states include:

- **R** → Running
- **S** → Sleeping
- **T** → Stopped
- **Z** → Zombie

---

## 🛡️ Cybersecurity Perspective

Process monitoring helps security professionals:

- Detect malware.
- Identify unauthorized applications.
- Investigate suspicious activity.
- Monitor CPU and memory usage.
- Respond to security incidents.

---

## ⭐ Best Practices

- Monitor running processes regularly.
- Investigate unknown processes.
- Avoid killing critical system processes.
- Use least privilege when managing processes.
- Keep system services updated.

---

## 💼 Real-World Example

A SOC Analyst notices unusually high CPU usage on a Linux server.

Using `top` and `ps`, they identify an unknown process consuming excessive resources. After investigation, the process is confirmed to be malicious and is terminated using the appropriate command.
