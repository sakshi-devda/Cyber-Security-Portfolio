# DHCP Analysis Practical

## 🎯 Objective

Capture and analyze DHCP traffic using Wireshark to observe the complete DORA (Discover, Offer, Request, Acknowledgment) process and understand how a device obtains an IP address automatically.

---

# Prerequisites

Before starting this practical, ensure that:

- Wireshark is installed.
- Npcap is installed.
- Your system is connected to a DHCP-enabled network.
- You have Administrator privileges.

---

# Step 1: Open Wireshark

Launch Wireshark.

Select your active network interface.

Examples:

- Wi-Fi
- Ethernet

---

# Step 2: Start Packet Capture

Double-click the active network interface to begin capturing packets.

---

# Step 3: Generate DHCP Traffic

Open **Command Prompt** as Administrator.

Release the current IP address:

```bash
ipconfig /release
```

Request a new IP address:

```bash
ipconfig /renew
```

These commands force the system to communicate with the DHCP server.

---

# Step 4: Stop Packet Capture

After the IP address is assigned successfully, stop the packet capture by clicking the **Red Stop** button.

---

# Step 5: Apply Display Filter

In the Display Filter bar, type:

```text
bootp
```

Press **Enter**.

Only DHCP packets will now be displayed.

---

# Step 6: Identify the DORA Process

Locate and verify the following DHCP messages:

- DHCP Discover
- DHCP Offer
- DHCP Request
- DHCP ACK

These four packets complete the DORA process.

---

# Step 7: Analyze DHCP Packet Details

Select each DHCP packet and expand the **Bootstrap Protocol (BOOTP)** section.

Observe the following fields:

- Transaction ID
- Client MAC Address
- Your (Assigned) IP Address
- DHCP Server IP Address
- DHCP Message Type
- Lease Time
- DHCP Options

---

# Step 8: Verify the Assigned IP Address

Open Command Prompt and run:

```bash
ipconfig
```

Verify that the assigned IP address matches the one received from the DHCP server.

---

# Expected Result

You should successfully observe:

- DHCP Discover
- DHCP Offer
- DHCP Request
- DHCP ACK
- Assigned IP Address
- DHCP Server Information
- Lease Time

# Observation

- The client successfully initiated the DHCP process by sending a Discover message.
- The DHCP server responded with an available IP address in the Offer message.
- The client requested the offered IP address.
- The server acknowledged the request and assigned the IP address.
- The complete DORA process was successfully captured and analyzed.

---

# Conclusion

In this practical, DHCP communication was successfully captured and analyzed using Wireshark. The complete DORA process was observed, demonstrating how a DHCP server automatically assigns an IP address and other network configuration settings to a client. This practical is useful for troubleshooting IP assignment issues and understanding DHCP communication in real-world networks.
