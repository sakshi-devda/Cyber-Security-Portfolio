# Export Objects - Theory

## 1. Introduction

Wireshark Export Objects is a feature that allows analysts to extract files and objects transferred through network traffic from a packet capture (PCAP) file.

It helps in recovering files exchanged between systems during communication.


## 2. Why Export Objects is Important

Network traffic may contain:

- Documents
- Images
- Executable files
- Configuration files
- Web content

Export Objects helps security analysts analyze these transferred files.


## 3. Supported Protocols

Wireshark can export objects from protocols such as:

- HTTP
- SMB
- TFTP
- DICOM


Commonly used:

### HTTP

Used to extract files downloaded or uploaded through web traffic.

### SMB

Used to analyze files transferred through Windows file sharing.

### TFTP

Used for simple file transfers.


## 4. Export Objects Working

Process:

1. Network communication occurs.
2. Packets capture the transferred data.
3. Wireshark reconstructs the data stream.
4. Objects are extracted and saved.


## 5. HTTP Object Extraction

HTTP traffic can contain:

- HTML files
- Images
- Scripts
- Downloads


Example:

A user downloads a file from a website.

Wireshark can recover that file from captured packets.


## 6. SMB Object Extraction

SMB is commonly used for:

- File sharing
- Network storage
- Windows communication


Export Objects can help analyze files transferred through SMB sessions.


## 7. Security Importance

Export Objects is useful for:

- Malware investigation
- File recovery
- Data leakage analysis
- Suspicious file detection
- Incident response


## 8. Malware Analysis Use

Attackers may transfer malicious files over the network.

Analysts can:

1. Extract the file.
2. Calculate its hash.
3. Analyze it in a safe environment.
4. Determine if it is malicious.


## 9. Limitations

Export Objects may not work when:

- Traffic is encrypted
- Required packets are missing
- Protocol is not supported
- Capture is incomplete


## 10. Encrypted Traffic

Protocols like HTTPS encrypt transferred data.

Wireshark cannot directly export readable objects unless decryption keys are available.


## 11. Conclusion

Wireshark Export Objects is a powerful feature for extracting transferred files from packet captures.

It is widely used in:
- Digital forensics
- Network investigations
- Malware analysis
- Incident response
