# DNS Analysis Practical

## 🎯 Objective

Capture and analyze DNS traffic using Wireshark to understand how domain names are resolved into IP addresses.

---

# Prerequisites

Before starting this practical, ensure that:

- Wireshark is installed.
- Npcap is installed.
- Your system is connected to the Internet.
- You have Administrator privileges.

---

# Step 1: Open Wireshark

Launch Wireshark and select your active network interface.

Examples:

- Wi-Fi
- Ethernet

---

# Step 2: Start Packet Capture

Double-click the active network interface to begin capturing packets.

---

# Step 3: Generate DNS Traffic

Open your web browser and visit any website.

Examples:

- https://www.google.com
- https://www.github.com
- https://www.wikipedia.org

Alternatively, open Command Prompt and run:

```bash
nslookup google.com
```

This generates DNS Query and DNS Response packets.

---

# Step 4: Stop Packet Capture

After a few seconds, stop the packet capture by clicking the **Red Stop** button.

---

# Step 5: Apply Display Filter

In the Display Filter bar, type:

```text
dns
```

Press **Enter**.

Only DNS packets will be displayed.

---

# Step 6: Identify the DNS Query

Locate the first DNS packet.

Verify the following information:

- Source IP Address
- Destination IP Address
- Requested Domain Name
- Query Type (A, AAAA, MX, etc.)

---

# Step 7: Identify the DNS Response

Locate the DNS Response packet.

Verify:

- Returned IP Address
- Response Code
- Record Type
- Time Taken

---

# Step 8: Inspect Packet Details

Expand the following sections:

- Ethernet II
- Internet Protocol (IP)
- User Datagram Protocol (UDP)
- Domain Name System (DNS)

Observe the available information in each protocol layer.

---

# Expected Result

You should successfully identify:

- DNS Query
- DNS Response
- Requested Domain Name
- Returned IP Address
- DNS Record Type
