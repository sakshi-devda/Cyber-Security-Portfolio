# Display Filters Theory

## 🎯 Objective

Understand what Display Filters are, how they work, why they are important, and how they help analyze specific network traffic in Wireshark.

---

# What are Display Filters?

Display Filters are used to display only the packets that match a specific condition without removing or modifying the original captured packets.

They help users quickly find relevant packets from thousands of captured packets.

Display Filters work only after packets have been captured.

---

# Why are Display Filters Needed?

A packet capture may contain thousands or even millions of packets.

Analyzing every packet manually is difficult and time-consuming.

Display Filters allow us to focus only on the packets we need.

Example:

Instead of viewing all packets, we can display only:

- HTTP traffic
- DNS traffic
- ICMP packets
- TCP packets
- Packets from a specific IP address

---

# How Display Filters Work

When a filter is applied:

1. Wireshark checks every captured packet.
2. It compares each packet with the filter expression.
3. Matching packets are displayed.
4. Non-matching packets are temporarily hidden.

No packets are deleted from the capture file.

---

# Advantages of Display Filters

- Faster packet analysis
- Easy troubleshooting
- Focus on specific protocols
- Identify suspicious traffic quickly
- Reduce unnecessary packet visibility

---

# Display Filters vs Capture Filters

| Display Filters | Capture Filters |
|-----------------|-----------------|
| Applied after packet capture | Applied before packet capture |
| Do not discard packets | Ignore unwanted packets during capture |
| Flexible and easy to modify | Cannot be changed after capture starts |
| Used for packet analysis | Used to reduce captured traffic |

# Common Display Filters

### Display all TCP packets

```text
tcp
```

---

### Display all UDP packets

```text
udp
```

---

### Display DNS packets

```text
dns
```

---

### Display HTTP packets

```text
http
```

---

### Display HTTPS (TLS) packets

```text
tls
```

---

### Display ICMP packets

```text
icmp
```

---

### Display ARP packets

```text
arp
```

---

### Display packets from a specific IP address

```text
ip.addr == 192.168.1.10
```

---

### Display packets sent from a source IP

```text
ip.src == 192.168.1.10
```

---

### Display packets sent to a destination IP

```text
ip.dst == 192.168.1.10
```

---

# Filter Expression

A Display Filter is written using a filter expression.

Example:

```text
tcp.port == 80
```

This filter displays only TCP packets using port 80.

---

# Combining Filters

Multiple filters can be combined using logical operators.

### AND

```text
tcp && ip.addr == 192.168.1.10
```

---

### OR

```text
http || dns
```

---

### NOT

```text
!icmp
```
