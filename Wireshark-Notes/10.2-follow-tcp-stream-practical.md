# Follow TCP Stream Practical

## 🎯 Objective

Learn how to use the Follow TCP Stream feature in Wireshark to reconstruct and analyze complete TCP conversations between a client and a server.

---

# Prerequisites

Before starting this practical, ensure that:

- Wireshark is installed.
- Npcap is installed.
- Your system is connected to the Internet.
- You have Administrator privileges.

> **Note:** For better results, use an HTTP website (such as `http://neverssl.com`) because HTTP traffic is not encrypted and can be viewed easily.

---

# Step 1: Open Wireshark

Launch Wireshark.

Select your active network interface.

Examples:

- Wi-Fi
- Ethernet

---

# Step 2: Start Packet Capture

Double-click the active network interface to begin capturing packets.

---

# Step 3: Generate TCP Traffic

Open your web browser.

Visit an HTTP website.

Example:

```
http://neverssl.com
```

Wait until the webpage loads completely.

---

# Step 4: Stop Packet Capture

Click the **Red Stop** button to stop capturing packets.

---

# Step 5: Apply Display Filter

In the Display Filter bar, type:

```text
http
```

Press **Enter**.

Only HTTP packets will now be displayed.

---

# Step 6: Open Follow TCP Stream

Select any HTTP packet.

Right-click the packet.

Choose:

```
Follow → TCP Stream
```

A new window displaying the complete TCP conversation will open.

---

# Step 7: Analyze the TCP Stream

Observe the following information:

- HTTP Request
- HTTP Response
- Request Method
- Host Name
- Requested Resource
- Response Headers
- HTML Content (if available)

---

# Step 8: Identify the Stream Number

At the top of the Follow TCP Stream window, note the **Stream Number**.

If multiple TCP sessions exist, switch between them using the stream selector.

---

# Step 9: Close the Window

After reviewing the conversation, close the Follow TCP Stream window.

Continue analyzing other packets if required.

---

# Expected Result

You should successfully observe:

- Complete TCP Conversation
- HTTP Request
- HTTP Response
- Request Headers
- Response Headers
- Stream Number

# Observation

- Wireshark successfully reconstructed the complete TCP conversation.
- The entire HTTP Request and Response were displayed in a single window.
- The Stream Number uniquely identified the TCP session.
- Follow TCP Stream made it easier to analyze application-layer communication compared to inspecting individual packets.

---

# Conclusion

In this practical, the Follow TCP Stream feature was used to reconstruct a complete TCP conversation. This simplified the analysis of client-server communication by displaying all related packets in a single readable stream. Follow TCP Stream is an essential feature for troubleshooting, network analysis, digital forensics, and cybersecurity investigations.
