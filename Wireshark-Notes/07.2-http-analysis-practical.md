# HTTP Analysis Practical

## 🎯 Objective

Capture and analyze HTTP traffic using Wireshark to understand HTTP requests, responses, methods, status codes, and headers.

---

# Prerequisites

Before starting this practical, ensure that:

- Wireshark is installed.
- Npcap is installed.
- Your system is connected to the Internet.
- You have Administrator privileges.

> **Note:** Most modern websites use HTTPS instead of HTTP. To capture HTTP packets, you may need to visit an HTTP website (for example, `http://neverssl.com`) or use a local web server.

---

# Step 1: Open Wireshark

Launch Wireshark.

Select your active network interface.

Examples:

- Wi-Fi
- Ethernet

---

# Step 2: Start Packet Capture

Double-click the active network interface to begin capturing packets.

---

# Step 3: Generate HTTP Traffic

Open your browser and visit an HTTP website.

Example:

```
http://neverssl.com
```

Wait until the webpage loads completely.

---

# Step 4: Stop Packet Capture

Click the **Red Stop** button to stop capturing packets.

---

# Step 5: Apply Display Filter

In the Display Filter bar, type:

```text
http
```

Press **Enter**.

Only HTTP packets should now be displayed.

---

# Step 6: Analyze the HTTP Request

Select an HTTP Request packet.

Expand the **Hypertext Transfer Protocol** section.

Observe:

- Request Method
- Request URI
- Host
- HTTP Version
- User-Agent

---

# Step 7: Analyze the HTTP Response

Select the corresponding HTTP Response packet.

Observe:

- Status Code
- Status Message
- Content-Type
- Content-Length
- Server

---

# Step 8: Inspect Packet Details

Expand the following protocol layers:

- Ethernet II
- Internet Protocol (IP)
- Transmission Control Protocol (TCP)
- Hypertext Transfer Protocol (HTTP)

Review the information available at each layer.

---

# Expected Result

You should successfully identify:

- HTTP Request
- HTTP Response
- Request Method
- Requested URL
- Status Code
- Response Headers

# Observation

- The browser sent an HTTP Request to the web server.
- The server responded with an HTTP Response.
- The request method, URL, status code, and headers were visible in Wireshark.
- HTTP traffic can be read because it is transmitted in plain text.

---

# Conclusion

In this practical, HTTP traffic was successfully captured and analyzed using Wireshark. The HTTP Request and Response packets were examined to understand how web browsers communicate with web servers. This practical demonstrates why HTTP traffic is easy to inspect and why HTTPS is preferred for secure communication.
