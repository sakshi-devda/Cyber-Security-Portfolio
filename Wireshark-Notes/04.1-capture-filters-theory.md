# Capture Filters Theory

## 🎯 Objective

Understand what Capture Filters are, how they work, why they are used, and how they help capture only the required network traffic.

---

# What are Capture Filters?

Capture Filters are used to capture only the packets that match a specified condition. Unlike Display Filters, they work before the packet is captured.

This helps reduce unnecessary network traffic and keeps the capture file smaller and easier to analyze.

---

# Why are Capture Filters Needed?

On busy networks, thousands of packets are transmitted every second.

Capturing all packets may:

- Consume more storage space.
- Make packet analysis difficult.
- Reduce system performance.
- Increase the time required for investigation.

Capture Filters solve this problem by collecting only the traffic you need.

---

# How Capture Filters Work

The packet capture process follows these steps:

1. Select a network interface.
2. Apply a Capture Filter.
3. Start packet capture.
4. Wireshark captures only the packets that match the filter.
5. Stop the capture.
6. Analyze the captured packets.

Packets that do not match the filter are not captured.

---

# Capture Filters vs Display Filters

| Capture Filters | Display Filters |
|-----------------|-----------------|
| Applied before packet capture | Applied after packet capture |
| Capture only matching packets | Show matching packets from an existing capture |
| Reduce file size | Do not reduce file size |
| Cannot be changed while capturing | Can be changed at any time |
| Use BPF (Berkeley Packet Filter) syntax | Use Wireshark Display Filter syntax |

# Common Capture Filters

## Capture TCP Traffic

```text
tcp
```

Captures only TCP packets.

---

## Capture UDP Traffic

```text
udp
```

Captures only UDP packets.

---

## Capture ICMP Traffic

```text
icmp
```

Captures only ICMP packets.

---

## Capture ARP Traffic

```text
arp
```

Captures only ARP packets.

---

## Capture Traffic from a Specific Host

```text
host 192.168.1.10
```

Captures packets sent to or received from the specified host.

---

## Capture Traffic from a Source Host

```text
src host 192.168.1.10
```

Captures packets originating from the specified host.

---

## Capture Traffic to a Destination Host

```text
dst host 192.168.1.10
```

Captures packets destined for the specified host.

---

## Capture a Specific Port

```text
port 80
```

Captures packets using port 80.

---

## Capture Source Port

```text
src port 443
```

Captures packets originating from source port 443.

---

## Capture Destination Port

```text
dst port 53
```

Captures packets destined for port 53.

---

# Logical Operators

Capture Filters support logical operators.

## AND

```text
tcp and port 80
```

---

## OR

```text
tcp or udp
```

---

## NOT

```text
not arp
```

# Advantages

- Reduces unnecessary packet capture.
- Saves storage space.
- Improves analysis speed.
- Reduces CPU and memory usage.
- Makes troubleshooting easier.

---

# Limitations

- Packets that are not captured cannot be recovered.
- Filters cannot be modified once the capture has started.
- Incorrect filters may result in missing important traffic.

---

# Capture Filters in Cyber Security

Capture Filters are useful for:

- Malware Analysis
- Network Monitoring
- Incident Response
- Threat Hunting
- SOC Operations
- Traffic Investigation
- Performance Analysis

---

# Best Practices

- Plan your filter before starting the capture.
- Capture only the required traffic.
- Test your filter if possible.
- Save important packet captures.
- Use simple filters whenever possible.

# Summary

Capture Filters allow Wireshark to capture only the required network traffic before the capture process begins. They reduce unnecessary data collection, improve performance, and make packet analysis faster and more efficient. Understanding Capture Filters is essential for effective network monitoring and cybersecurity investigations.
