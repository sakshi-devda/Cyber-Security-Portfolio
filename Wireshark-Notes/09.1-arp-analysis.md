# ARP Analysis

## 🎯 Objective

Understand how the Address Resolution Protocol (ARP) works, how MAC addresses are resolved from IP addresses, and how ARP packets are analyzed using Wireshark.

---

# What is ARP?

ARP (Address Resolution Protocol) is a Network Layer support protocol used to map an IPv4 address to a MAC (Media Access Control) address within a local network.

Devices use ARP before sending data on a LAN because communication at the Data Link Layer requires a destination MAC address.

---

# Why is ARP Important?

When a device knows the destination IP address but does not know its MAC address, it cannot send an Ethernet frame.

ARP helps the device discover the MAC address associated with the destination IP address.

Without ARP, communication within a local network would not be possible.

---

# How ARP Works

The ARP process follows these steps:

1. The sender checks its ARP Cache.
2. If the MAC address is found, communication begins immediately.
3. If not found, the sender broadcasts an ARP Request.
4. The destination device sends an ARP Reply containing its MAC address.
5. The sender stores the MAC address in its ARP Cache for future communication.

---

# ARP Communication Flow

```
Client
   │
   ▼
ARP Request (Broadcast)
"Who has 192.168.1.1?"
   │
   ▼
Network
   │
   ▼
Destination Device
   │
   ▼
ARP Reply
"192.168.1.1 is at AA:BB:CC:DD:EE:FF"
   │
   ▼
Client Updates ARP Cache
```

---

# ARP Request

An ARP Request is broadcast to every device on the local network.

Example:

```
Who has 192.168.1.1?
Tell 192.168.1.100
```

Every device receives the request, but only the device with the matching IP address replies.

---

# ARP Reply

The destination device sends an ARP Reply directly to the sender.

Example:

```
192.168.1.1 is at
AA:BB:CC:DD:EE:FF
```

The sender stores this MAC address in its ARP Cache.

---

# ARP Cache

An ARP Cache is a temporary table maintained by the operating system.

It stores recently resolved IP-to-MAC address mappings to reduce unnecessary ARP Requests.

On Windows, view the ARP Cache using:

```bash
arp -a
```

---

# ARP Packet Fields

An ARP packet contains important fields such as:

- Hardware Type
- Protocol Type
- Hardware Size
- Protocol Size
- Opcode
- Sender MAC Address
- Sender IP Address
- Target MAC Address
- Target IP Address

---

# ARP in Wireshark

ARP packets can be displayed using the following Display Filter:

```text
arp
```

Wireshark allows analysts to inspect:

- Sender MAC Address
- Sender IP Address
- Target MAC Address
- Target IP Address
- Request and Reply Messages

---

# ARP in Cyber Security

ARP analysis helps security professionals to:

- Troubleshoot LAN connectivity.
- Detect duplicate IP addresses.
- Identify ARP Spoofing attacks.
- Investigate Man-in-the-Middle (MITM) attacks.
- Monitor unusual ARP traffic.
- Support incident response investigations.

---

# Advantages

- Automatically resolves MAC addresses.
- Enables communication within a local network.
- Reduces manual network configuration.
- Improves communication efficiency using ARP Cache.

---

# Limitations

- Works only within a local network.
- Does not support IPv6 (IPv6 uses Neighbor Discovery Protocol).
- Vulnerable to ARP Spoofing and ARP Poisoning attacks.
