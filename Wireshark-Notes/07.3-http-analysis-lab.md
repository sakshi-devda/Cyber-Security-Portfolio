# HTTP Analysis Lab

## 🎯 Lab Objective

Capture and analyze HTTP traffic using Wireshark to identify HTTP requests, responses, methods, headers, and status codes during a web browsing session.

---

# Scenario

You are working as a Junior SOC Analyst.

A user reports that they visited an unknown website before experiencing unusual browser behavior. Your task is to analyze the HTTP traffic and determine what resources were requested and how the server responded.

---

# Lab Requirements

- Wireshark Installed
- Npcap Installed
- Internet Connection
- Administrator Privileges

> **Note:** Since most modern websites use HTTPS, use an HTTP website such as `http://neverssl.com` or a local web server for this lab.

---

The HTTP Request and HTTP Response packets were successfully identified. The request method, requested resource, response status code, and HTTP headers were analyzed. This demonstrated how a browser communicates with a web server using the HTTP protocol.

# What I Learned

- How HTTP Request and Response packets appear in Wireshark.
- How to identify HTTP methods and status codes.
- How to inspect HTTP headers.
- Why HTTP traffic is readable in plain text.
- Why HTTPS is preferred for secure communication.

# Conclusion
