# Packet Capturing

## 📌 Objective

Understand what packet capturing is, how it works, different capture modes, and why packet capturing is important in networking and cybersecurity.

---

# What is Packet Capturing?

Packet capturing is the process of collecting network packets as they travel between devices on a network.

A packet capture tool like Wireshark intercepts these packets and displays them for detailed analysis.

Packet capturing helps us understand how devices communicate and identify problems or suspicious activities within a network.

---

# What is a Network Packet?

A network packet is a small unit of data transmitted over a network.

Whenever you:

- Visit a website
- Send an email
- Watch a video
- Download a file
- Use online applications

your device exchanges thousands of packets with other devices.

Each packet contains important information such as:

- Source Address
- Destination Address
- Protocol
- Payload (Actual Data)
- Control Information

---

# How Packet Capturing Works

The packet capturing process generally follows these steps:

1. Select a network interface.
2. Start packet capture.
3. The network interface receives packets.
4. Wireshark copies the packets before they reach the operating system.
5. Captured packets are displayed in real time.
6. The user stops the capture and begins analysis.

---

# Promiscuous Mode

Promiscuous Mode allows a network interface card (NIC) to capture all packets it receives, not just packets addressed to the local device.

This mode is commonly used for:

- Network Monitoring
- Troubleshooting
- Security Analysis
- Packet Inspection

---

# Monitor Mode

Monitor Mode is mainly used on wireless (Wi-Fi) networks.

Unlike Promiscuous Mode, Monitor Mode allows the wireless adapter to capture all wireless frames within its range, including traffic that is not intended for the local device.

Monitor Mode is commonly used for:

- Wireless Security Audits
- Wi-Fi Troubleshooting
- Wireless Packet Analysis

> **Note:** Not all wireless adapters support Monitor Mode.

---

# Active vs Passive Packet Capture

## Active Packet Capture

In active packet capture, the analyst generates network traffic intentionally.

Examples:

- Ping a website
- Open a browser
- Download a file

This helps capture specific packets for analysis.

---

## Passive Packet Capture

In passive packet capture, Wireshark simply listens to existing network traffic without generating any additional traffic.

This method is commonly used in:

- SOC Monitoring
- Network Monitoring
- Incident Response

---

# Why Packet Capturing is Important

Packet capturing helps administrators and security professionals to:

- Troubleshoot network problems
- Detect malicious traffic
- Investigate cyber attacks
- Analyze application communication
- Identify protocol errors
- Verify firewall rules
- Monitor network performance

---

# Packet Capture Workflow

A typical packet capture workflow is:

```

Select Interface
↓
Start Capture
↓
Generate or Observe Traffic
↓
Stop Capture
↓
Apply Filters
↓
Analyze Packets
↓
Save Capture File

```

---

# Challenges in Packet Capturing

Some challenges include:

- Large amount of traffic
- Encrypted communication
- High-speed networks
- Packet loss
- Unsupported network adapters

---

# Packet Capturing in Cyber Security

Packet capturing is an essential skill for cybersecurity professionals.

It is widely used in:

- Malware Analysis
- Threat Hunting
- Digital Forensics
- Incident Response
- Network Security Monitoring
- Penetration Testing
- Security Operations Centers (SOC)

---

# Advantages

- Provides real-time network visibility
- Helps detect abnormal behavior
- Supports protocol analysis
- Useful for troubleshooting
- Essential for forensic investigations

---

# Limitations

- Cannot automatically identify every attack
- HTTPS traffic is encrypted
- Large capture files require filtering
- Requires sufficient permissions to capture packets
