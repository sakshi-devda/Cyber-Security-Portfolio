# DNS Analysis Theory

## 🎯 Objective

Understand how the Domain Name System (DNS) works, how DNS packets are analyzed in Wireshark, and why DNS analysis is important in networking and cybersecurity.

---

# What is DNS?

The Domain Name System (DNS) is a protocol that translates human-readable domain names into IP addresses.

For example, instead of remembering:

```
142.250.183.110
```

we simply type:

```
www.google.com
```

DNS finds the corresponding IP address so that the browser can communicate with the correct server.

---

# Why is DNS Important?

Computers communicate using IP addresses, while humans prefer easy-to-remember domain names.

DNS acts as a translator between domain names and IP addresses.

Without DNS, users would need to remember the IP address of every website they want to visit.

---

# How DNS Works

When you enter a website address in your browser:

1. The browser checks its local DNS cache.
2. If the record is not found, the request is sent to a DNS server.
3. The DNS server searches for the corresponding IP address.
4. The DNS server sends the IP address back to the client.
5. The browser uses the IP address to connect to the web server.

---

# DNS Resolution Process

```
User
   │
   ▼
Browser
   │
   ▼
DNS Resolver
   │
   ▼
DNS Server
   │
   ▼
Returns IP Address
   │
   ▼
Browser Connects to Website
```

---

# DNS Packet Components

A DNS packet contains several important fields:

- Transaction ID
- Flags
- Questions
- Answers
- Authority Records
- Additional Records

These fields help identify and process DNS requests and responses.

---

# DNS Query

A DNS Query is a request sent by the client asking for the IP address of a domain.

Example:

```
What is the IP address of www.google.com?
```

---

# DNS Response

A DNS Response is the reply sent by the DNS server containing the requested information.

Example:

```
www.google.com

142.250.xxx.xxx
```

---

# Common DNS Record Types

| Record | Purpose |
|---------|----------|
| A | Maps a domain name to an IPv4 address |
| AAAA | Maps a domain name to an IPv6 address |
| CNAME | Alias for another domain |
| MX | Mail server information |
| NS | Name server information |
| TXT | Additional text information |
| PTR | Reverse DNS lookup |

---

# DNS in Wireshark

DNS packets can be displayed using the following Display Filter:

```text
dns
```

This filter shows both DNS queries and DNS responses.

---

# Information Available in DNS Packets

Wireshark allows you to view:

- Source IP
- Destination IP
- Domain Name
- Query Type
- Response Code
- Time Taken
- Returned IP Address

---

# DNS in Cyber Security

DNS analysis is extremely important because attackers often misuse DNS.

Security professionals analyze DNS traffic to:

- Detect malware communication
- Identify phishing domains
- Detect DNS tunneling
- Investigate suspicious domains
- Monitor unusual DNS requests
- Detect Command and Control (C2) communication

---

# Advantages of DNS Analysis

- Helps troubleshoot website connectivity issues.
- Detects suspicious domain requests.
- Identifies malicious DNS traffic.
- Assists in malware investigations.
- Supports incident response activities.

---

# Limitations

- DNS over HTTPS (DoH) encrypts DNS traffic, making analysis more difficult.
- Cached DNS records may reduce visible queries.
- Large networks can generate thousands of DNS requests.
