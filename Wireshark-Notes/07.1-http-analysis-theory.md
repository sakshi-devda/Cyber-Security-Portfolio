# HTTP Analysis

## 🎯 Objective

Understand how HTTP works, analyze HTTP packets in Wireshark, and learn the importance of HTTP traffic in networking and cybersecurity.

---

# What is HTTP?

HTTP (Hypertext Transfer Protocol) is an application layer protocol used for communication between a client (such as a web browser) and a web server.

Whenever you open a website, your browser sends an HTTP request to the server, and the server responds with the requested content.

HTTP is the foundation of data communication on the World Wide Web.

---

# Why is HTTP Important?

HTTP enables users to:

- Browse websites
- Download web pages
- Submit forms
- Upload files
- Access web applications
- Communicate with web servers

Without HTTP, web browsers would not be able to retrieve website content.

---

# How HTTP Works

The communication process follows these steps:

1. The client sends an HTTP Request.
2. The web server receives the request.
3. The server processes the request.
4. The server sends an HTTP Response.
5. The browser displays the received content.

---

# HTTP Communication Flow

```
Client (Browser)
       │
       ▼
HTTP Request
       │
       ▼
Web Server
       │
       ▼
HTTP Response
       │
       ▼
Browser Displays the Web Page
```

---

# HTTP Request

An HTTP Request is sent by the client to request a resource from the server.

A request typically contains:

- Request Method
- URL
- HTTP Version
- Headers
- Body (optional)

Example:

```
GET /index.html HTTP/1.1
Host: example.com
```

---

# HTTP Response

The server replies with an HTTP Response.

A response contains:

- Status Code
- Status Message
- Headers
- Response Body

Example:

```
HTTP/1.1 200 OK
Content-Type: text/html
```

---

# Common HTTP Methods

| Method | Purpose |
|----------|----------|
| GET | Retrieve data from the server |
| POST | Send data to the server |
| PUT | Update an existing resource |
| DELETE | Remove a resource |
| HEAD | Retrieve only the headers |
| OPTIONS | Display supported communication methods |

---

# Common HTTP Status Codes

| Status Code | Meaning |
|--------------|---------|
| 200 OK | Request completed successfully |
| 201 Created | Resource created successfully |
| 301 Moved Permanently | Resource has moved permanently |
| 302 Found | Temporary redirection |
| 400 Bad Request | Invalid client request |
| 401 Unauthorized | Authentication required |
| 403 Forbidden | Access denied |
| 404 Not Found | Requested resource not found |
| 500 Internal Server Error | Server-side error |

---

# HTTP Headers

Headers provide additional information about the request or response.

Common headers include:

- Host
- User-Agent
- Accept
- Content-Type
- Content-Length
- Cookie
- Server

---

# HTTP in Wireshark

HTTP packets can be displayed using the following Display Filter:

```text
http
```

Wireshark allows analysts to inspect:

- Request Method
- Requested URL
- Host Name
- Status Code
- Headers
- Response Data

---

# HTTP vs HTTPS

| HTTP | HTTPS |
|------|-------|
| Data is transmitted in plain text | Data is encrypted using TLS/SSL |
| Default Port: 80 | Default Port: 443 |
| Less secure | More secure |
| Data can be intercepted | Data is protected during transmission |

---

# HTTP in Cyber Security

HTTP analysis helps security professionals to:

- Detect suspicious web traffic.
- Investigate malicious downloads.
- Analyze phishing websites.
- Identify insecure web communication.
- Investigate web-based attacks.
- Monitor client-server communication.

---

# Advantages

- Simple and widely supported.
- Easy to analyze.
- Fast communication.
- Foundation of web browsing.

---

# Limitations

- HTTP traffic is not encrypted.
- Sensitive data can be intercepted.
- Vulnerable to attacks such as Man-in-the-Middle (MITM).
- Modern websites generally use HTTPS instead of HTTP.

# Summary

HTTP is an application layer protocol used for communication between clients and web servers. It follows a request-response model and enables web browsing, file transfers, and web application communication. Wireshark allows analysts to inspect HTTP packets, making HTTP analysis an essential skill for troubleshooting and cybersecurity investigations.
