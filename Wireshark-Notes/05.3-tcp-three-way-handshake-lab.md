# TCP Three-Way Handshake Lab

## 🎯 Lab Objective

Capture live network traffic and identify the complete TCP Three-Way Handshake between a client and a server using Wireshark.

---

# Scenario

You are working as a Junior SOC Analyst.

A user reports that they are unable to access a website. Before investigating further, your first task is to verify whether the TCP connection between the client and the server is being established successfully.

You decide to capture the network traffic using Wireshark and analyze the TCP Three-Way Handshake.

---

# Lab Requirements

- Wireshark Installed
- Npcap Installed
- Internet Connection
- Administrator Privileges

# Lab Analysis

During packet analysis, the following observations were made:

- The client initiated communication by sending a SYN packet.
- The server acknowledged the request using a SYN-ACK packet.
- The client completed the handshake with an ACK packet.
- The TCP connection was successfully established before any application data was transmitted.

This lab demonstrated the complete TCP Three-Way Handshake using live packet capture in Wireshark.
