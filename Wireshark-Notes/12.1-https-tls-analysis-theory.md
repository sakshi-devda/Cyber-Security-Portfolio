# HTTPS & TLS Analysis

## 🎯 Objective

Understand how HTTPS and TLS work, how the TLS Handshake establishes a secure connection, and how HTTPS traffic can be analyzed using Wireshark.

---

# What is HTTPS?

HTTPS (HyperText Transfer Protocol Secure) is the secure version of HTTP.

It uses TLS (Transport Layer Security) to encrypt communication between a client and a web server.

Unlike HTTP, HTTPS protects sensitive information from being intercepted during transmission.

---

# Why is HTTPS Important?

HTTPS provides secure communication over the Internet.

It protects:

- Usernames
- Passwords
- Banking Information
- Personal Data
- Cookies
- API Requests

Almost every modern website uses HTTPS to ensure secure communication.

---

# HTTP vs HTTPS

| HTTP | HTTPS |
|------|-------|
| Unencrypted | Encrypted |
| Uses Port 80 | Uses Port 443 |
| Less Secure | Highly Secure |
| Data can be intercepted | Data is protected using TLS |

---

# What is TLS?

TLS (Transport Layer Security) is a cryptographic protocol used to secure communication over a network.

TLS provides:

- Confidentiality
- Integrity
- Authentication

TLS is the successor to SSL (Secure Sockets Layer).

---

# Why is TLS Important?

Without TLS:

- Passwords could be stolen.
- Sensitive information could be modified.
- Attackers could intercept network traffic.

TLS protects communication from these threats.

---

# TLS Handshake

Before encrypted communication begins, the client and server perform a TLS Handshake.

The handshake establishes:

- Secure Connection
- Encryption Algorithm
- Session Keys
- Server Authentication

---

# TLS Handshake Process

```
Client
   │
Client Hello
   ▼
Server
   │
Server Hello
   ▼
Certificate
   ▼
Key Exchange
   ▼
Finished
   ▼
Encrypted Communication
```

---

# Client Hello

The client sends:

- Supported TLS Versions
- Supported Cipher Suites
- Random Number
- Extensions

---

# Server Hello

The server responds with:

- Selected TLS Version
- Selected Cipher Suite
- Random Number

---

# Digital Certificate

The server sends its Digital Certificate.

The certificate contains information such as:

- Domain Name
- Public Key
- Certificate Authority (CA)
- Expiration Date
- Digital Signature

The client verifies the certificate before continuing the connection.

---

# Cipher Suite

A Cipher Suite defines:

- Key Exchange Algorithm
- Encryption Algorithm
- Authentication Algorithm

Both the client and server must agree on the same Cipher Suite.

---

# Session Keys

After the handshake, both the client and server generate a shared session key.

This key is used to encrypt and decrypt all communication.

---

# Encrypted Communication

Once the TLS Handshake is complete:

- HTTP traffic becomes HTTPS.
- Data is encrypted.
- Wireshark cannot read the application data unless decryption keys are available.

---

# HTTPS in Wireshark

Display Filter:

```text
tls
```

Common information visible:

- Client Hello
- Server Hello
- Certificate
- Application Data
- TLS Version
- Cipher Suite
- Server Name (SNI)

---

# Cyber Security Importance

HTTPS analysis helps security professionals to:

- Verify secure communication.
- Inspect TLS Handshakes.
- Validate server certificates.
- Detect weak TLS versions.
- Identify insecure cipher suites.
- Troubleshoot HTTPS connectivity issues.

---

# Advantages

- Encrypts network traffic.
- Protects sensitive information.
- Authenticates servers.
- Prevents data tampering.
- Improves user trust.

---

# Limitations

- Encrypted application data cannot be viewed without decryption keys.
- Improper certificate configuration may cause connection failures.
- Older TLS versions are less secure.

# Summary

HTTPS is the secure version of HTTP that uses TLS to encrypt communication between clients and servers. During the TLS Handshake, both parties authenticate each other and establish secure session keys before exchanging encrypted data. Wireshark allows analysts to inspect the handshake process, certificates, and encryption details, making HTTPS and TLS analysis an essential skill for networking and cybersecurity professionals.
