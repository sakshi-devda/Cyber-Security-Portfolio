# DHCP Analysis

## 🎯 Objective

Understand how the Dynamic Host Configuration Protocol (DHCP) works, analyze the DHCP DORA process using Wireshark, and learn its importance in networking and cybersecurity.

---

# What is DHCP?

DHCP (Dynamic Host Configuration Protocol) is an Application Layer protocol that automatically assigns IP addresses and other network configuration settings to devices on a network.

Instead of manually configuring every device, DHCP automatically provides:

- IP Address
- Subnet Mask
- Default Gateway
- DNS Server
- Lease Time

This simplifies network management and reduces configuration errors.

---

# Why is DHCP Important?

Without DHCP, every device would need to be configured manually.

DHCP helps to:

- Automatically assign IP addresses.
- Prevent IP address conflicts.
- Reduce administrative effort.
- Simplify network management.

---

# How DHCP Works

DHCP follows a four-step process called **DORA**.

```
Client
   │
   ▼
DHCP Discover
   │
   ▼
DHCP Server
   │
   ▼
DHCP Offer
   │
   ▼
Client
   │
   ▼
DHCP Request
   │
   ▼
DHCP Server
   │
   ▼
DHCP Acknowledgment (ACK)
```

---

# DORA Process

## 1. DHCP Discover

The client broadcasts a DHCP Discover message to locate available DHCP servers.

---

## 2. DHCP Offer

The DHCP server replies with an available IP address and network configuration.

---

## 3. DHCP Request

The client requests the offered IP address from the DHCP server.

---

## 4. DHCP Acknowledgment (ACK)

The DHCP server confirms the assignment and leases the IP address to the client.

The client can now communicate on the network.

---

# DHCP Packet Information

A DHCP packet contains information such as:

- Client MAC Address
- Assigned IP Address
- DHCP Server IP
- Lease Time
- Transaction ID
- DHCP Message Type

---

# DHCP in Wireshark

Display Filter:

```text
bootp
```

> **Note:** Although the protocol is DHCP, Wireshark commonly displays DHCP packets using the `bootp` filter because DHCP is based on the BOOTP protocol.

---

# Practical

## Step 1

Open Wireshark.

Select the active network interface.

---

## Step 2

Start packet capture.

---

## Step 3

Generate DHCP traffic.

Open Command Prompt and run:

```bash
ipconfig /release

ipconfig /renew
```

This forces the system to request a new IP address from the DHCP server.

---

## Step 4

Stop packet capture.

---

## Step 5

Apply the filter:

```text
bootp
```

---

## Step 6

Locate the following packets:

- DHCP Discover
- DHCP Offer
- DHCP Request
- DHCP ACK

# Cyber Security Importance

DHCP analysis helps security professionals to:

- Troubleshoot IP assignment issues.
- Detect Rogue DHCP Servers.
- Investigate network connectivity problems.
- Monitor unauthorized devices.
- Support incident response.

# Summary

DHCP automatically assigns network configuration to devices using the DORA process: Discover, Offer, Request, and Acknowledgment. Wireshark makes it easy to analyze DHCP traffic, troubleshoot IP assignment issues, and investigate network-related problems. DHCP analysis is an essential skill for network administrators and cybersecurity professionals.
