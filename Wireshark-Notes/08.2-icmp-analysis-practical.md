# ICMP Analysis Practical

## 🎯 Objective

Capture and analyze ICMP packets using Wireshark to understand how the Ping command works and how ICMP Echo Request and Echo Reply packets are exchanged.

---

# Prerequisites

Before starting this practical, ensure that:

- Wireshark is installed.
- Npcap is installed.
- Your system is connected to the Internet.
- You have Administrator privileges.

---

# Step 1: Open Wireshark

Launch Wireshark.

Select your active network interface.

Examples:

- Wi-Fi
- Ethernet

---

# Step 2: Start Packet Capture

Double-click the active network interface to begin capturing packets.

---

# Step 3: Generate ICMP Traffic

Open **Command Prompt** and run the following command:

```bash
ping google.com
```

Alternatively, you can ping any reachable IP address.

Wait until all ping requests have completed.

---

# Step 4: Stop Packet Capture

Click the **Red Stop** button to stop capturing packets.

---

# Step 5: Apply Display Filter

In the Display Filter bar, type:

```text
icmp
```

Press **Enter**.

Only ICMP packets should now be displayed.

---

# Step 6: Analyze the Echo Request

Select an **Echo (ping) Request** packet.

Expand the **Internet Control Message Protocol** section.

Observe:

- Type
- Code
- Identifier
- Sequence Number
- Source IP
- Destination IP

---

# Step 7: Analyze the Echo Reply

Select the corresponding **Echo (ping) Reply** packet.

Observe:

- Type
- Code
- Identifier
- Sequence Number
- Response Time

Verify that the Identifier and Sequence Number match the Echo Request.

---

# Step 8: Inspect Packet Details

Expand the following protocol layers:

- Ethernet II
- Internet Protocol (IP)
- Internet Control Message Protocol (ICMP)

Review the information available in each layer.

---

# Expected Result

You should successfully identify:

- ICMP Echo Request
- ICMP Echo Reply
- Source IP Address
- Destination IP Address
- Sequence Number
- Response Time

# Observation

- The client sent ICMP Echo Request packets to verify connectivity.
- The destination host responded with ICMP Echo Reply packets.
- The request and reply shared the same Identifier and Sequence Number.
- The response time indicated the network latency between the client and the destination.
