# Wireshark Statistics - Practical

## 1. Introduction

Wireshark Statistics tools help analyze captured network traffic by providing detailed information about packets, protocols, endpoints, and communication patterns.

In this practical, we will explore different Statistics features in Wireshark.


## 2. Opening Statistics Menu

Steps:

1. Open Wireshark
2. Open a capture file or start live capture
3. Click on:

   Analyze → Statistics


From here, different statistical tools can be accessed.


## 3. Capture Summary

Steps:

1. Go to:

   Statistics → Capture File Properties

2. View information like:

- Total packets
- Capture duration
- Packet rate
- File size


Purpose:
- Quickly understand the size and duration of captured traffic.


## 4. Protocol Hierarchy Analysis

Steps:

1. Go to:

   Statistics → Protocol Hierarchy

2. Analyze:

- Protocol names
- Packet count
- Percentage of traffic
- Bytes transferred


Example:

TCP may show higher traffic because many applications use TCP connections.


Security Use:
- Identify unknown or unexpected protocols.


## 5. Endpoint Analysis

Steps:

1. Go to:

   Statistics → Endpoints

2. Select:

- IPv4
- IPv6
- Ethernet


Analyze:

- IP addresses
- Packet count
- Bytes transferred


Security Use:
- Find active hosts in the network.


## 6. Conversation Analysis

Steps:

1. Go to:

   Statistics → Conversations

2. Select protocol type:

- TCP
- UDP
- Ethernet


Analyze:

- Source address
- Destination address
- Packets
- Bytes


Security Use:
- Identify important communication sessions.


## 7. I/O Graph Analysis

Steps:

1. Go to:

   Statistics → I/O Graph


2. Observe:

- Packet rate
- Traffic spikes
- Network activity over time


Security Use:
- Detect unusual traffic patterns.


## 8. Flow Graph Analysis

Steps:

1. Go to:

   Statistics → Flow Graph

2. Select:

- General Flow
- TCP Flow


Observe:

- Communication sequence
- Request and response packets


Security Use:
- Understand client-server communication.
