# HTTPS & TLS Analysis Practical

## 🎯 Objective

Capture and analyze HTTPS traffic using Wireshark to observe the TLS Handshake, inspect server certificates, and understand how encrypted communication is established.

---

# Prerequisites

Before starting this practical, ensure that:

- Wireshark is installed.
- Npcap is installed.
- Your system is connected to the Internet.
- You have Administrator privileges.

---

# Step 1: Open Wireshark

Launch Wireshark.

Select your active network interface.

Examples:

- Wi-Fi
- Ethernet

---

# Step 2: Start Packet Capture

Double-click the active network interface to begin capturing packets.

---

# Step 3: Generate HTTPS Traffic

Open any web browser.

Visit an HTTPS website.

Examples:

```
https://www.google.com

https://github.com

https://openai.com
```

Wait until the website loads completely.

---

# Step 4: Stop Packet Capture

Click the **Red Stop** button to stop packet capture.

---

# Step 5: Apply Display Filter

In the Display Filter bar, type:

```text
tls
```

Press **Enter**.

Only TLS packets will be displayed.

---

# Step 6: Analyze the TLS Handshake

Locate the following handshake messages:

- Client Hello
- Server Hello
- Certificate
- Application Data

Expand each packet to view its details.

---

# Step 7: Verify the Certificate

Open the **Certificate** packet.

Observe the following details:

- Subject Name
- Issuer
- Valid From
- Valid To
- Public Key Algorithm
- Signature Algorithm

---

# Step 8: Verify TLS Information

Check the following information:

- TLS Version
- Cipher Suite
- Server Name (SNI)

---

# Step 9: Observe Encrypted Data

Select an **Application Data** packet.

Notice that the payload is encrypted and cannot be read without the required decryption keys.

---

# Expected Result

You should successfully observe:

- Client Hello
- Server Hello
- Server Certificate
- TLS Version
- Cipher Suite
- Encrypted Application Data

# Observation

- The client initiated communication by sending a **Client Hello** message.
- The server responded with a **Server Hello** and its digital certificate.
- The client verified the certificate and completed the TLS Handshake.
- After the handshake, all application data was encrypted.
- Wireshark displayed handshake information but could not read the encrypted HTTPS payload.

---

# Conclusion

In this practical, HTTPS traffic was successfully captured and analyzed using Wireshark. The TLS Handshake was examined, including the Client Hello, Server Hello, and Certificate exchange. The practical demonstrated how TLS establishes a secure connection and protects application data through encryption, making HTTPS communication secure over the Internet.
