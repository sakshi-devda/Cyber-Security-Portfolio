# ICMP Analysis

## 🎯 Objective

Understand the Internet Control Message Protocol (ICMP), how it works, its message types, and how ICMP packets are analyzed using Wireshark for network troubleshooting and cybersecurity investigations.

---

# What is ICMP?

ICMP (Internet Control Message Protocol) is a Network Layer protocol used by devices to send error messages and operational information during network communication.

Unlike TCP and UDP, ICMP does not transport application data. Instead, it helps diagnose network connectivity and report network-related issues.

---

# Why is ICMP Important?

ICMP helps administrators and security professionals to:

- Verify network connectivity.
- Diagnose communication problems.
- Detect unreachable hosts.
- Measure network latency.
- Troubleshoot routing issues.

---

# How ICMP Works

When a device needs to verify connectivity, it sends an ICMP Echo Request packet to another device.

If the destination device is reachable, it replies with an ICMP Echo Reply packet.

This communication confirms that the destination host is available on the network.

---

# ICMP Communication Flow

```
Client
   │
   ▼
ICMP Echo Request
   │
   ▼
Destination Host
   │
   ▼
ICMP Echo Reply
   │
   ▼
Client
```

---

# Common ICMP Message Types

| Type | Description |
|------|-------------|
| 0 | Echo Reply |
| 3 | Destination Unreachable |
| 5 | Redirect Message |
| 8 | Echo Request |
| 11 | Time Exceeded |

---

# Echo Request

An Echo Request is sent by the source device to verify whether the destination device is reachable.

Example:

```
Client
   │
   ▼
Echo Request
```

---

# Echo Reply

If the destination device is reachable, it responds with an Echo Reply.

Example:

```
Destination
   │
   ▼
Echo Reply
```

---

# Ping

The `ping` command uses ICMP Echo Request and Echo Reply packets to test connectivity between two devices.

Example:

```bash
ping google.com
```

A successful ping indicates that the destination host is reachable.

---

# Traceroute

Traceroute uses ICMP messages (or UDP/TCP depending on the operating system and implementation) to identify the path packets take from the source to the destination.

It helps identify where communication delays or failures occur.

---

# ICMP Packet Fields

An ICMP packet contains several important fields, including:

- Type
- Code
- Checksum
- Identifier
- Sequence Number
- Data

These fields help devices process ICMP messages correctly.

---

# ICMP in Wireshark

ICMP packets can be displayed using the following Display Filter:

```text
icmp
```

Wireshark allows analysts to inspect:

- Echo Request
- Echo Reply
- Source IP
- Destination IP
- Type
- Code
- Sequence Number
- Response Time

---

# ICMP in Cyber Security

ICMP analysis helps security professionals to:

- Troubleshoot network connectivity.
- Detect network scanning.
- Identify unreachable systems.
- Investigate suspicious ICMP traffic.
- Detect ICMP-based attacks such as Ping Floods.
- Support incident response investigations.

---

# Advantages

- Simple connectivity testing.
- Useful for troubleshooting.
- Measures network latency.
- Helps diagnose routing problems.
- Supported by almost all network devices.

---

# Limitations

- Some firewalls block ICMP traffic.
- ICMP does not provide secure communication.
- Can be abused for network reconnaissance and denial-of-service attacks.
