# TCP Three-Way Handshake Practical

## 🎯 Objective

Capture and identify the TCP Three-Way Handshake (SYN, SYN-ACK, ACK) using Wireshark and understand how a TCP connection is established before data transmission.

---

# Prerequisites

Before starting this practical, ensure that:

- Wireshark is installed.
- Npcap is installed.
- Your system is connected to the Internet.
- You have Administrator privileges.

---

# Step 1: Open Wireshark

Launch Wireshark and select your active network interface.

Examples:

- Wi-Fi
- Ethernet

---

# Step 2: Start Packet Capture

Double-click the active network interface to begin capturing packets.

---

# Step 3: Generate TCP Traffic

Open any website in your browser.

Examples:

- https://google.com
- https://github.com
- https://openai.com

Opening a website will generate a TCP connection.

---

# Step 4: Stop Packet Capture

After the webpage loads successfully:

- Click the **Red Stop** button.

---

# Step 5: Apply Display Filter

In the Display Filter bar, type:

```text
tcp
```

Press **Enter**.

Only TCP packets will now be displayed.

---

# Step 6: Locate the TCP Handshake

Look for the following sequence:

```
SYN
↓

SYN, ACK
↓

ACK
```

These three packets indicate that a TCP connection has been successfully established.

---

# Step 7: Inspect the SYN Packet

Select the first packet.

Expand:

```
Transmission Control Protocol
```

Observe:

- Source Port
- Destination Port
- Sequence Number
- SYN Flag

---

# Step 8: Inspect the SYN-ACK Packet

Select the second packet.

Observe:

- SYN Flag
- ACK Flag
- Sequence Number
- Acknowledgment Number

---

# Step 9: Inspect the ACK Packet

Select the third packet.

Observe:

- ACK Flag
- Updated Sequence Number
- Updated Acknowledgment Number

The connection is now established.

---

# Expected Result

You should successfully identify:

- SYN Packet
- SYN-ACK Packet
- ACK Packet

This confirms that the TCP Three-Way Handshake was completed successfully.

# Observation

- The client initiated the connection by sending a SYN packet.
- The server responded with a SYN-ACK packet.
- The client completed the connection by sending an ACK packet.
- After the handshake, data transmission began.

---

# Conclusion

In this practical, the TCP Three-Way Handshake was successfully captured and analyzed using Wireshark. The SYN, SYN-ACK, and ACK packets confirmed the successful establishment of a TCP connection. Understanding this process is essential for network troubleshooting, protocol analysis, and cybersecurity investigations.
