# TCP Three-Way Handshake Theory

## 🎯 Objective

Understand how a TCP connection is established using the Three-Way Handshake, why it is necessary, and how it is analyzed in Wireshark.

---

# What is TCP?

TCP (Transmission Control Protocol) is a connection-oriented protocol that provides reliable, ordered, and error-checked delivery of data between devices on a network.

Before any data is transmitted, TCP establishes a connection between the client and the server.

---

# What is the TCP Three-Way Handshake?

The TCP Three-Way Handshake is the process used to establish a reliable connection between a client and a server before exchanging data.

It consists of three steps:

1. SYN
2. SYN-ACK
3. ACK

After these three steps, the connection is successfully established.

---

# Why is the Three-Way Handshake Required?

The handshake ensures that:

- Both devices are reachable.
- Both devices are ready to communicate.
- Initial sequence numbers are synchronized.
- A reliable connection is established before data transfer.

---

# Step 1: SYN (Synchronize)

The client initiates the connection by sending a SYN packet to the server.

This packet contains the client's Initial Sequence Number (ISN).

Example:

```
Client → Server
SYN
Seq = 1000
```

The client is requesting to establish a connection.

---

# Step 2: SYN-ACK (Synchronize Acknowledge)

The server receives the SYN request and responds with a SYN-ACK packet.

This packet:

- Acknowledges the client's sequence number.
- Sends the server's own sequence number.

Example:

```
Server → Client
SYN, ACK
Seq = 5000
Ack = 1001
```

The server is ready to establish the connection.

---

# Step 3: ACK (Acknowledge)

The client sends the final ACK packet.

This confirms that the server's response has been received successfully.

Example:

```
Client → Server
ACK
Seq = 1001
Ack = 5001
```

After this step, the TCP connection is established.

---

# Three-Way Handshake Flow

```
Client                           Server

  SYN  ------------------------->

       <------------------- SYN-ACK

  ACK  ------------------------->
```

Connection Established

---

# Sequence Number

A Sequence Number identifies the position of data being transmitted.

It helps TCP:

- Track transmitted data.
- Detect missing packets.
- Reassemble packets in the correct order.

---

# Acknowledgment Number

The Acknowledgment Number tells the sender which packet is expected next.

It confirms successful packet delivery.

---

# Why TCP is Reliable

TCP provides reliability through:

- Three-Way Handshake
- Sequence Numbers
- Acknowledgment Numbers
- Error Detection
- Retransmission of Lost Packets
- Flow Control
- Congestion Control

---

# Where is TCP Handshake Used?

Every time you:

- Open a website
- Login to a server
- Access email
- Connect through SSH
- Browse securely using HTTPS

a TCP Three-Way Handshake occurs before data transfer begins.

---

# TCP Handshake in Wireshark

In Wireshark, the handshake can be identified by observing three packets:

| Packet | TCP Flag |
|---------|----------|
| Packet 1 | SYN |
| Packet 2 | SYN, ACK |
| Packet 3 | ACK |

These packets appear before the actual application data.

---

# Cyber Security Importance

Analyzing the TCP Handshake helps security professionals to:

- Troubleshoot connection issues.
- Detect failed connections.
- Identify SYN Flood attacks.
- Investigate network anomalies.
- Analyze suspicious communication.

---

# Advantages

- Reliable communication
- Ordered packet delivery
- Error recovery
- Connection verification
- Stable data transmission

---

# Limitations

- Additional connection setup time.
- Higher overhead than UDP.
- Less suitable for real-time applications like live video or gaming.

# Summary

The TCP Three-Way Handshake is the process of establishing a reliable connection between a client and a server before data transmission. It consists of three packets—SYN, SYN-ACK, and ACK—which synchronize communication, verify connectivity, and ensure both devices are ready to exchange data. Understanding this process is fundamental for networking, troubleshooting, and cybersecurity analysis.
