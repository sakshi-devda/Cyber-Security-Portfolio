# DHCP Analysis

## 🎯 Objective

Understand how DHCP works, how devices automatically obtain IP addresses, and how DHCP packets can be analyzed using Wireshark for network troubleshooting and cybersecurity investigations.

---

# What is DHCP?

DHCP (Dynamic Host Configuration Protocol) is an Application Layer protocol that automatically assigns IP addresses and other network configuration settings to devices connected to a network.

Instead of manually configuring every device, DHCP allows a DHCP Server to automatically provide network settings whenever a new device joins the network.

---

# Why is DHCP Important?

Without DHCP, every device on a network would require manual configuration.

DHCP helps to:

- Automatically assign IP addresses.
- Reduce manual configuration.
- Prevent IP address conflicts.
- Simplify network administration.
- Manage IP address allocation efficiently.

---

# DHCP Components

A DHCP environment consists of the following components:

### DHCP Client

The device requesting network configuration from the DHCP Server.

Examples:

- Laptop
- Desktop
- Mobile Phone

---

### DHCP Server

The device responsible for assigning IP addresses and other network settings.

---

### DHCP Relay Agent

A router or network device that forwards DHCP messages between clients and DHCP servers located on different networks.

---

### DHCP Scope

A range of IP addresses that the DHCP Server can assign to clients.

Example:

```
192.168.1.100
        ↓
192.168.1.200
```

---

# DHCP Ports

DHCP uses the User Datagram Protocol (UDP).

| Device | Port |
|---------|------|
| DHCP Server | UDP 67 |
| DHCP Client | UDP 68 |

---

# How DHCP Works

DHCP follows a four-step process called **DORA**.

```
Client
   │
Discover
   ▼
DHCP Server
   │
Offer
   ▼
Client
   │
Request
   ▼
DHCP Server
   │
ACK
   ▼
Client Receives IP Address
```

---

# DORA Process

## 1. DHCP Discover

The client broadcasts a DHCP Discover message to locate available DHCP servers.

---

## 2. DHCP Offer

The DHCP Server replies with an available IP address and network configuration.

---

## 3. DHCP Request

The client requests the offered IP address from the DHCP Server.

---

## 4. DHCP ACK (Acknowledgment)

The DHCP Server confirms the request and leases the IP address to the client.

The client can now communicate on the network.

---

# DHCP Messages

Common DHCP message types include:

- DHCP Discover
- DHCP Offer
- DHCP Request
- DHCP ACK
- DHCP NAK
- DHCP Release
- DHCP Inform

---

# DHCP Lease

A DHCP Lease is the period during which a client is allowed to use an assigned IP address.

Important lease stages include:

- Lease Time
- Renewal (T1)
- Rebinding (T2)
- Lease Expiration

If the lease expires without renewal, the client must request a new IP address.

---

# DHCP Packet Fields

A DHCP packet contains several important fields:

- Transaction ID
- Client MAC Address
- Your (Assigned) IP Address
- Server IP Address
- Lease Time
- DHCP Message Type
- DHCP Options

---

# DHCP in Wireshark

DHCP packets can be displayed using the following Display Filter:

```text
bootp
```

> **Note:** Although the protocol is DHCP, Wireshark uses the `bootp` display filter because DHCP is an extension of the BOOTP protocol.

# DHCP in Cyber Security

DHCP analysis helps security professionals to:

- Troubleshoot IP address assignment issues.
- Detect Rogue DHCP Servers.
- Identify unauthorized devices.
- Investigate network connectivity problems.
- Detect DHCP-based attacks.

---

# Advantages

- Automatic IP address assignment.
- Reduces manual configuration.
- Prevents IP conflicts.
- Easy network management.
- Efficient IP address utilization.

---

# Limitations

- Dependency on the DHCP Server.
- A Rogue DHCP Server can provide incorrect network settings.
- DHCP messages are not encrypted.

# Summary

DHCP is an Application Layer protocol that automatically assigns IP addresses and network configuration to devices. It follows the DORA process—Discover, Offer, Request, and Acknowledgment—to provide seamless network connectivity. Wireshark allows analysts to inspect DHCP communication, troubleshoot IP assignment issues, and detect unauthorized DHCP activity, making DHCP analysis an essential networking and cybersecurity skill.
