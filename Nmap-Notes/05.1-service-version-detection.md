# Service & Version Detection

## 📌 What is Service & Version Detection?

Service and Version Detection is a feature of Nmap that identifies which services are running on open ports and determines their software versions.

Instead of just telling you that a port is open, Nmap also tells you *what is actually running behind that port*.

---

## 🎯 Why is Service Detection Important?

Service detection helps security professionals to:

- Identify running applications on open ports.
- Detect outdated or vulnerable software versions.
- Perform vulnerability assessments.
- Improve network security visibility.
- Support penetration testing and auditing.

---

# 🔍 How It Works

Nmap sends special probes to open ports and analyzes the responses (banners, protocol behavior, etc.) to identify:

- Service name (e.g., SSH, HTTP, FTP)
- Software version (e.g., OpenSSH 8.2)
- Additional service details

---

# ⚙️ Nmap Command for Service Detection

## Basic Command

```bash
nmap -sV scanme.nmap.org
```

---

## 📌 What does `-sV` do?

- Enables service version detection.
- Probes open ports.
- Extracts service banners.
- Identifies software versions.

---

# 📊 Example Output (Simplified)

```
22/tcp open  ssh     OpenSSH 8.4
80/tcp open  http    Apache httpd 2.4.41
443/tcp open https   nginx 1.18.0
```

---

# 🧠 What This Output Means

| Port | Service | Version |
|------|--------|--------|
| 22 | SSH | OpenSSH 8.4 |
| 80 | HTTP | Apache 2.4.41 |
| 443 | HTTPS | Nginx 1.18.0 |

---

# 🔎 Intensity Levels

Nmap allows different intensity levels for version detection:

```bash
nmap -sV --version-intensity 5 scanme.nmap.org
```

| Level | Description |
|------|-------------|
| 0 | Light scanning |
| 2–5 | Balanced scanning |
| 7–9 | Aggressive detection |

---

# 🛡️ Why Attackers Care About Version Detection

Attackers use version detection to:

- Find outdated software.
- Identify known vulnerabilities (CVEs).
- Select exploit techniques.
- Map attack surface.

---

# 🛡️ Why Defenders Use It

Security teams use it to:

- Patch vulnerable services.
- Monitor exposed software.
- Harden systems.
- Audit network services.

---

# 💼 Real-World Example

A company server is scanned and shows:

```
Apache httpd 2.2.15
```

This version is outdated and has known vulnerabilities.

The security team upgrades it to a secure version to prevent exploitation.

---

# ⚠️ Limitations

- Firewalls may block version detection.
- Some services do not expose banners.
- Fake or modified banners can mislead detection.
- Accuracy depends on network conditions.

---

# ⭐ Best Practices

- Always use `-sV` after identifying open ports.
- Combine with `-O` (OS detection) for deeper analysis.
- Validate results manually when possible.
- Keep software updated to avoid exposure.
