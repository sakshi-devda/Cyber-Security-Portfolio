# Output Formats Lab

## 🎯 Objective

In this lab, you will generate Nmap scan reports in different output formats, compare the generated files, and understand how each format is used in professional cybersecurity environments.

---

# Lab Environment

- Operating System: Kali Linux
- Tool: Nmap
- Target: scanme.nmap.org or a TryHackMe machine

---

# Task 1: Generate a Normal Report

## Command

```bash
nmap -oN report.txt scanme.nmap.org
```

### Verify

```bash
cat report.txt
```

### Observe

- Host information
- Open ports
- Running services

📸 **Screenshot:** Normal report

---

# Task 2: Generate an XML Report

## Command

```bash
nmap -oX report.xml scanme.nmap.org
```

### Verify

```bash
cat report.xml
```

### Observe

- XML structure
- Host details
- Port information

📸 **Screenshot:** XML report

---

# Task 3: Generate a Grepable Report

## Command

```bash
nmap -oG report.gnmap scanme.nmap.org
```

### Verify

```bash
cat report.gnmap
```

### Observe

- Grep-friendly format
- Host status
- Port summary

📸 **Screenshot:** Grepable report

---

# Task 4: Generate Reports in All Formats

## Command

```bash
nmap -oA final_report scanme.nmap.org
```

### Verify

```bash
ls
```

### Expected Files

```
final_report.nmap
final_report.xml
final_report.gnmap
```

📸 **Screenshot:** Generated files

---

# Task 5: Generate a Professional Scan Report

## Command

```bash
nmap -A -oA professional_scan scanme.nmap.org
```

### Observe

- OS information
- Service versions
- NSE results
- Traceroute

📸 **Screenshot:** Professional report

---

# Task 6: Compare Output Formats

| Format | Human Readable | Automation | Best Use |
|---------|----------------|------------|----------|
| `.nmap` | ✅ Yes | ❌ No | Documentation |
| `.xml` | ⚠️ Limited | ✅ Yes | Security tools |
| `.gnmap` | ⚠️ Limited | ✅ Yes | Filtering and scripting |

---

# Analysis Questions

1. Which output format is easiest for humans to read?
2. Which format is best for automation?
3. Which format would you attach to a penetration testing report?
4. Which format is useful for importing into security tools?
5. Why is `-oA` commonly used in professional environments?

---

# Challenge

Perform the following scan:

```bash
sudo nmap -A -sV -O -oA network_audit scanme.nmap.org
```

After the scan:

- Review every generated file.
- Compare the information in each format.
- Identify open ports, services, and OS details.
- Save all files for future reference.

---

# Best Practices

- Always save important scans.
- Use descriptive report names.
- Store reports securely.
- Remove sensitive reports when they are no longer needed.
- Keep scan reports as part of security documentation.
