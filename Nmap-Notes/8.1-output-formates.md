# Output Formats

## 📌 What are Output Formats?

Output formats in Nmap allow scan results to be saved in different file formats for reporting, analysis, automation, and future reference.

Instead of displaying results only in the terminal, Nmap can export scan data into files that can be shared, searched, or imported into other security tools.

---

## 🎯 Why are Output Formats Important?

Output formats help security professionals to:

- Save scan results.
- Generate security reports.
- Compare scan results over time.
- Share findings with team members.
- Import results into other tools.
- Maintain proper documentation.

---

# 📂 Types of Output Formats

## 1. Normal Output (-oN)

Saves scan results in a human-readable text format.

### Command

```bash
nmap -oN scan.txt scanme.nmap.org
```

### File Extension

```
.txt
```

### Best For

- Reading scan reports manually.
- Documentation.

---

## 2. XML Output (-oX)

Saves scan results in XML format.

### Command

```bash
nmap -oX scan.xml scanme.nmap.org
```

### File Extension

```
.xml
```

### Best For

- Automation.
- Importing into security tools.
- Report generation.

---

## 3. Grepable Output (-oG)

Stores results in a format that is easy to search using tools like `grep`.

### Command

```bash
nmap -oG scan.grep scanme.nmap.org
```

### File Extension

```
.grep
```

### Best For

- Filtering results.
- Command-line analysis.
- Legacy scripts.

> Note: Grepable output is considered legacy and may not include all modern Nmap features.

---

## 4. All Formats (-oA)

Saves results in multiple formats at the same time.

### Command

```bash
nmap -oA scan scanme.nmap.org
```

This creates:

```
scan.nmap
scan.xml
scan.gnmap
```

### Best For

- Complete reporting.
- Long-term documentation.
- Professional security assessments.

---

# 📊 Output Format Comparison

| Format | Option | Best Use |
|--------|--------|----------|
| Normal | `-oN` | Human-readable reports |
| XML | `-oX` | Automation and integration |
| Grepable | `-oG` | Command-line filtering |
| All Formats | `-oA` | Save all output types |

---

# 🛡️ Output Formats in Cybersecurity

Security teams use output files to:

- Create penetration testing reports.
- Document security findings.
- Track network changes.
- Archive scan results.
- Share scan data with other teams.

---

# 💼 Real-World Example

A penetration tester scans a company's network using:

```bash
nmap -sV -oA internal_scan 192.168.1.0/24
```

The generated files are:

- `internal_scan.nmap`
- `internal_scan.xml`
- `internal_scan.gnmap`

The `.xml` file is imported into another security tool for further analysis, while the `.nmap` file is included in the final assessment report.

---

# ⚠️ Limitations

- Output files may contain sensitive information.
- Reports should be stored securely.
- XML files can become large for extensive scans.
- Scan results should be protected from unauthorized access.

---

# ⭐ Best Practices

- Save important scans using `-oA`.
- Store scan reports securely.
- Use meaningful file names.
- Review reports before sharing.
- Archive reports for future comparison.
