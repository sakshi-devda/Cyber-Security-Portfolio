# Port Scanning Lab

## 🎯 Objective

In this lab, you will perform different types of port scans using Nmap, identify open ports, analyze scan results, and understand the security implications of exposed services.

---

# Lab Environment

- Operating System: Kali Linux
- Tool: Nmap
- Target: scanme.nmap.org or a TryHackMe Machine

---

# Task 1: Perform a Default Scan

## Command

```bash
nmap scanme.nmap.org
```

### Observe

- Is the host up?
- Which ports are open?
- Which services are running?

📸 **Screenshot:** Default Scan

---

# Task 2: Scan Specific Ports

## Command

```bash
nmap -p 22,80,443 scanme.nmap.org
```

### Observe

- Which of these ports are open?
- Which services are associated with them?

📸 **Screenshot:** Specific Port Scan

---

# Task 3: Scan All TCP Ports

## Command

```bash
nmap -p- scanme.nmap.org
```

### Observe

- How many ports are open?
- Are there any unexpected services?

📸 **Screenshot:** Full Port Scan

---

# Task 4: Scan Multiple Hosts

## Command

```bash
nmap 192.168.1.10 192.168.1.20
```

> Replace the IP addresses with valid hosts in your lab environment.

### Observe

- Which hosts are online?
- Do they have different open ports?

📸 **Screenshot:** Multiple Host Scan

# Task 5: Analyze the Results

Record your observations.

| Port | Service | Status | Security Risk |
|------|---------|--------|---------------|
| 22 | SSH | Open | Low (if properly secured) |
| 80 | HTTP | Open | Medium (if outdated) |
| 443 | HTTPS | Open | Low (with valid TLS configuration) |

Add more entries based on your scan results.

---

# Analysis Questions

1. Which host had the highest number of open ports?
2. Which services were running?
3. Were there any unexpected open ports?
4. Which services could pose a security risk?
5. How would you reduce the attack surface?

---

# Challenge

Try scanning:

- A local virtual machine.
- A TryHackMe target.
- A Metasploitable machine (if available).

Compare the scan results and identify the differences.

---

# Best Practices

- Scan only systems you own or are authorized to test.
- Document your findings.
- Investigate unnecessary open ports.
- Recommend closing or filtering unused services.
- Repeat scans after applying security changes to verify improvements.

# Key Takeaway

Port scanning is one of the most important phases of network reconnaissance. By identifying open ports and the services running on them, security professionals can detect potential vulnerabilities, reduce the attack surface, and strengthen the overall security of a network.
