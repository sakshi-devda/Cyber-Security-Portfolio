# Scan Techniques Lab

## 🎯 Objective

In this lab, you will perform multiple Nmap scan techniques on the same target, compare the results, and understand when each scan method should be used.

---

# Lab Environment

- Operating System: Kali Linux
- Tool: Nmap
- Target: scanme.nmap.org or a TryHackMe Machine

---

# Task 1: TCP Connect Scan

## Command

```bash
nmap -sT scanme.nmap.org
```

### Observe

- Scan duration
- Open ports
- Running services

📸 **Screenshot:** TCP Connect Scan

---

# Task 2: SYN Scan

## Command

```bash
sudo nmap -sS scanme.nmap.org
```

### Observe

- Compare the scan speed with the TCP Connect Scan.
- Are the detected ports the same?

# Analysis Questions

1. Which scan completed the fastest?
2. Which scan took the longest?
3. Which scan technique is the stealthiest?

# Challenge

Perform the same scans on:

- A TryHackMe machine.
- A local virtual machine.
- Another authorized host.

Compare the results and note any differences caused by the operating system or firewall configuration.

---

# What I Learned

- Different Nmap scan techniques serve different purposes.
- Scan speed and stealth vary depending on the method used.
- Firewalls can influence scan results.
- Choosing the right scan technique improves the effectiveness of network reconnaissance.

---

# Best Practices

- Use the least intrusive scan necessary.
- Obtain authorization before scanning.
- Compare results from multiple scan techniques.
- Document all observations and findings.
- Verify unusual results with additional scans.

---

# Key Takeaway

Selecting the appropriate Nmap scan technique depends on the assessment objective, target environment, and security controls in place. Understanding the strengths and limitations of each method is essential for effective network reconnaissance and security testing.
