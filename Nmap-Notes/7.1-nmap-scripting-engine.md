# Nmap Scripting Engine (NSE)

## 📌 What is NSE?

Nmap Scripting Engine (NSE) is a powerful feature of Nmap that allows users to run scripts to automate network scanning, vulnerability detection, and advanced reconnaissance tasks.

NSE is written in the Lua programming language and significantly extends Nmap’s functionality.

---

## 🎯 Why is NSE Important?

NSE helps security professionals to:

- Automate complex scanning tasks.
- Detect vulnerabilities.
- Perform advanced service enumeration.
- Gather detailed system information.
- Conduct security audits.
- Speed up penetration testing workflows.

---

# ⚙️ How NSE Works

NSE uses pre-written scripts that are executed against target systems during a scan.

Each script performs a specific function such as:

- Checking vulnerabilities
- Enumerating users
- Detecting services
- Brute force testing
- Gathering system information

---

# 📂 NSE Script Categories

## 1. Safe Scripts

- Do not harm the target system.
- Used for basic information gathering.

---

## 2. Default Scripts

- Run automatically with `-sC`.
- Provide useful general information.

---

## 3. Discovery Scripts

- Used to discover network information.
- Example: hostnames, services.

---

## 4. Vulnerability Scripts

- Check for known vulnerabilities.
- Very useful in security assessments.

---

## 5. Auth Scripts

- Test authentication mechanisms.
- Useful for brute force testing.

---

## 6. Intrusive Scripts

- May affect target system performance.
- Should be used carefully.

---

## 7. Exploit Scripts

- Attempt to exploit vulnerabilities.
- Used in penetration testing labs.

---

# ⚙️ Basic NSE Commands

## Default Script Scan

```bash
nmap -sC scanme.nmap.org
```

---

## Run Specific Script Category

```bash
nmap --script default scanme.nmap.org
```

---

## Run a Specific Script

```bash
nmap --script http-title scanme.nmap.org
```

---

## Vulnerability Scan Example

```bash
nmap --script vuln scanme.nmap.org
```

# 💼 Real-World Example

A security analyst runs:

```bash
nmap -sC -sV target.com
```

The scan reveals:
- Apache server version
- SSL misconfiguration
- Open SSH service

The analyst then investigates vulnerabilities based on script output.

---

# ⚠️ Limitations

- Some scripts may be intrusive.
- Requires careful usage in production environments.
- Can be detected by IDS/IPS systems.
- Not all scripts are safe for external networks.

---

# ⭐ Best Practices

- Use `-sC` for safe default scripts.
- Avoid intrusive scripts on production systems.
- Combine NSE with `-sV` and `-O`.
- Read script documentation before use.
- Run in authorized environments only.
