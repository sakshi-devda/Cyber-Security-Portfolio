# Service & Version Detection (Practical)

## 🎯 Objective

In this practical, you will learn how to identify running services and their versions using Nmap.

You will also understand how version detection helps in real-world security analysis.

---

# Lab Environment

- OS: Kali Linux
- Tool: Nmap
- Target: scanme.nmap.org or your lab machine

---

# Practical 1: Basic Service Detection

## Command

```bash
nmap -sV scanme.nmap.org
```

### Purpose

Detects running services and their versions on open ports.

### Expected Output

- Open ports
- Service names
- Software versions

📸 **Screenshot:** Basic `-sV` scan

---

# Practical 2: Verbose Service Detection

## Command

```bash
nmap -sV -v scanme.nmap.org
```

### Purpose

Provides more detailed output during scanning.

### Expected Output

- Live scan progress
- Detected services with versions

📸 **Screenshot:** Verbose scan output

---

# Practical 3: Aggressive Version Detection

## Command

```bash
nmap -sV --version-intensity 9 scanme.nmap.org
```

### Purpose

Performs deep probing to detect service versions more accurately.

### Expected Output

- More precise service version detection
- Slightly longer scan time

📸 **Screenshot:** Aggressive version scan

---

# Practical 4: Version Detection on Specific Ports

## Command

```bash
nmap -sV -p 22,80,443 scanme.nmap.org
```

### Purpose

Checks service versions only on selected ports.

📸 **Screenshot:** Specific port version detection

---

# Practical 5: Full Port + Version Scan

## Command

```bash
nmap -p- -sV scanme.nmap.org
```

### Purpose

Scans all TCP ports and detects services and versions.

⚠️ This scan may take longer than usual.

📸 **Screenshot:** Full port version scan

---

# Practical 6: Output Interpretation

Example output:

```
22/tcp open  ssh     OpenSSH 8.2
80/tcp open  http    Apache httpd 2.4.41
443/tcp open https   nginx 1.18.0
```

---

## 🧠 Analysis

| Port | Service | Version | Risk |
|------|--------|--------|------|
| 22 | SSH | OpenSSH 8.2 | Medium |
| 80 | HTTP | Apache 2.4.41 | High (if outdated) |
| 443 | HTTPS | Nginx 1.18.0 | Low |

---

# 🔎 Key Observations

- `-sV` identifies both service and version.
- Aggressive scanning improves accuracy.
- Full scans take more time.
- Some services may not reveal version details.

---

# 🛡️ Security Perspective

Service version detection is used to:

- Identify outdated software.
- Detect vulnerable services.
- Map attack surface.
- Support vulnerability scanning.

---

# 💼 Real-World Example

A security analyst scans a server and finds:

```
OpenSSH 6.6
Apache 2.2.15
```

Both versions are outdated and contain known vulnerabilities. The organization upgrades them to secure versions.

---

# ⭐ Best Practices

- Always combine with port scanning.
- Use `-sV` after identifying open ports.
- Verify results manually if needed.
- Avoid aggressive scans on production systems.
- Document all findings.
