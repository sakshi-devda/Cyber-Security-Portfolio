# Firewall Evasion Basics (Practical)

## 🎯 Objective

In this practical, you will safely explore how different Nmap options can be used in an authorized lab to evaluate firewall behavior and security controls.

---

# Lab Environment

- Operating System: Kali Linux
- Tool: Nmap
- Target: Authorized lab machine (TryHackMe, Hack The Box, or your own VM)

---

# Practical 1: Basic Firewall Test

## Command

```bash
nmap <target-ip>
```

### Purpose

Performs a standard scan to establish a baseline before comparing other scan options.

### Expected Output

- Host status
- Open ports
- Basic service information

📸 **Screenshot:** Basic scan

---

# Practical 2: Fragmented Packet Scan

## Command

```bash
sudo nmap -f <target-ip>
```

### Purpose

Tests how the target firewall handles fragmented packets in a controlled lab.

📸 **Screenshot:** Fragmentation scan

---

# Practical 3: Decoy Scan

## Command

```bash
sudo nmap -D RND:5 <target-ip>
```

### Purpose

Demonstrates how decoy addresses appear during an authorized assessment.

📸 **Screenshot:** Decoy scan

---

# Practical 4: Source Port Test

## Command

```bash
sudo nmap --source-port 53 <target-ip>
```

### Purpose

Evaluates firewall behavior when packets use a different source port.

📸 **Screenshot:** Source port test

# Practical 5: MAC Address Spoofing (Local Network Only)

## Command

```bash
sudo nmap --spoof-mac 00:11:22:33:44:55 <target-ip>
```

### Purpose

Tests MAC-based filtering in a local lab environment.

# Summary of Commands

| Command | Purpose |
|---------|---------|
| `nmap <target>` | Baseline scan |
| `-f` | Fragmented packet test |
| `-D RND:5` | Decoy scan |
| `--source-port 53` | Source port evaluation |
| `--spoof-mac` | MAC spoofing (local networks) |

---

# Key Observations

- Firewall behavior may differ depending on scan options.
- Modern firewalls often detect advanced scan techniques.
- Results vary based on firewall rules and network configuration.
- Not every technique changes the scan outcome.

---

# Security Perspective

These techniques help defenders:

- Validate firewall rules.
- Verify IDS/IPS alerts.
- Identify configuration weaknesses.
- Improve network security.

---

# Best Practices

- Test only on systems you are authorized to assess.
- Document every scan result.
- Compare findings across multiple scan methods.
- Coordinate testing with the network owner.
- Review firewall logs after testing.
