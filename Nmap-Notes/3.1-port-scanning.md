# Port Scanning

## 📌 What is Port Scanning?

Port Scanning is the process of checking the ports of a target system to determine which ports are open, closed, or filtered.

Nmap sends packets to different ports and analyzes the responses to identify the status of each port.

Port scanning is one of the most important techniques used in network administration, security auditing, and penetration testing.

---

## 🎯 Why is Port Scanning Important?

Port scanning helps to:

- Identify open ports.
- Discover running services.
- Detect unnecessary open ports.
- Identify potential security risks.
- Prepare for vulnerability assessments.
- Support penetration testing.

---

# 🌐 What is a Port?

A port is a logical communication endpoint used by applications and services to exchange data over a network.

Each service listens on a specific port number.

### Examples

| Port | Service |
|------|---------|
| 22 | SSH |
| 25 | SMTP |
| 53 | DNS |
| 80 | HTTP |
| 110 | POP3 |
| 143 | IMAP |
| 443 | HTTPS |

---

# 📊 Port States in Nmap

## 1. Open

The port is accepting connections and a service is running.

---

## 2. Closed

The port is reachable but no service is listening.

---

## 3. Filtered

A firewall or security device is blocking Nmap from determining the port status.

---

## 4. Unfiltered

The port is reachable, but Nmap cannot determine whether it is open or closed.

---

## 5. Open | Filtered

Nmap cannot determine whether the port is open or filtered.

---

## 6. Closed | Filtered

Nmap cannot determine whether the port is closed or filtered.

# 🛡️ Port Scanning in Cybersecurity

Cybersecurity professionals use port scanning to:

- Identify exposed services.
- Detect unauthorized services.
- Verify firewall rules.
- Assess network security.
- Perform reconnaissance before penetration testing.

---

# 💼 Real-World Example

A security analyst scans a web server and discovers:

- Port 22 (SSH) — Open
- Port 80 (HTTP) — Open
- Port 443 (HTTPS) — Open
- Port 3306 (MySQL) — Open

The analyst notices that MySQL is publicly accessible. Since database services should not normally be exposed to the Internet, the analyst recommends restricting access through firewall rules.

---

# ⭐ Best Practices

- Scan only authorized systems.
- Close unused ports.
- Restrict access to sensitive services.
- Regularly audit open ports.
- Monitor for unexpected services.
- Document scan results.
