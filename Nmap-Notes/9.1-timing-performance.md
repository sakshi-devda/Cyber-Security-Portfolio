# Timing & Performance

## 📌 What is Timing & Performance?

Timing and Performance options in Nmap control how fast or slow a scan is performed. These options help balance scan speed, accuracy, network load, and stealth.

By adjusting timing settings, security professionals can optimize scans for different environments.

---

# 🎯 Why is Timing & Performance Important?

Timing options help to:

- Reduce scan time.
- Avoid overwhelming the target network.
- Minimize detection by security systems.
- Improve scan efficiency.
- Adapt scans to different network conditions.

---

# ⏱️ Timing Templates

Nmap provides six timing templates.

## T0 – Paranoid

```bash
nmap -T0 scanme.nmap.org
```

### Description

- Extremely slow.
- Designed to avoid detection.
- Sends packets one at a time.

### Use Case

Highly monitored networks.

---

## T1 – Sneaky

```bash
nmap -T1 scanme.nmap.org
```

### Description

- Very slow.
- Lower chance of detection.
- Suitable for stealthy scanning.

---

## T2 – Polite

```bash
nmap -T2 scanme.nmap.org
```

### Description

- Reduces network load.
- Good for production environments.

---

## T3 – Normal (Default)

```bash
nmap -T3 scanme.nmap.org
```

### Description

- Default timing.
- Balanced speed and accuracy.

---

## T4 – Aggressive

```bash
nmap -T4 scanme.nmap.org
```

### Description

- Faster scanning.
- Commonly used on fast and stable networks.

---

## T5 – Insane

```bash
nmap -T5 scanme.nmap.org
```

### Description

- Extremely fast.
- May miss results on slow or unstable networks.

---

# 📊 Timing Template Comparison

| Template | Speed | Stealth | Recommended Use |
|----------|-------|----------|-----------------|
| T0 | Very Slow | Very High | Maximum stealth |
| T1 | Slow | High | Stealth scanning |
| T2 | Medium | Medium | Production networks |
| T3 | Normal | Medium | General scanning |
| T4 | Fast | Low | Internal networks, labs |
| T5 | Very Fast | Very Low | High-speed lab testing |

# ⚙️ Performance Options

## Scan Delay

Adds a delay between packets.

```bash
nmap --scan-delay 1s scanme.nmap.org
```

---

## Maximum Retries

Limits packet retransmissions.

```bash
nmap --max-retries 2 scanme.nmap.org
```

---

## Host Timeout

Stops scanning a host after a specified time.

```bash
nmap --host-timeout 2m scanme.nmap.org
```

---

## Minimum Rate

Sets the minimum packet transmission rate.

```bash
nmap --min-rate 1000 scanme.nmap.org
```

---

## Maximum Rate

Limits the maximum packet transmission rate.

```bash
nmap --max-rate 500 scanme.nmap.org
```

---

# 🛡️ Timing in Cybersecurity

Security professionals adjust timing to:

- Scan large networks efficiently.
- Avoid triggering IDS/IPS alerts.
- Reduce network congestion.
- Improve scan reliability.

---

# 💼 Real-World Example

An internal security team needs to scan 500 servers overnight.

Instead of using the default timing, they use:

```bash
nmap -T4 --min-rate 1000 192.168.1.0/24
```

This significantly reduces scan time while maintaining reliable results on a fast internal network.

---

# ⚠️ Limitations

- Very fast scans may miss open ports.
- Slow scans take much longer to complete.
- Network latency affects scan performance.
- Aggressive timing may trigger security systems.

---

# ⭐ Best Practices

- Use `-T3` for general scanning.
- Use `-T4` on trusted internal networks.
- Avoid `-T5` unless testing in a lab.
- Adjust timing based on network conditions.
- Verify important findings with additional scans.
