# Timing & Performance Lab

## 🎯 Objective

In this lab, you will compare different Nmap timing templates and performance options to understand their impact on scan speed, accuracy, and network load.

---

# Lab Environment

- Operating System: Kali Linux
- Tool: Nmap
- Target: scanme.nmap.org or a TryHackMe machine

---

# Task 1: Default Timing Scan

## Command

```bash
nmap scanme.nmap.org
```

### Observe

- Scan duration
- Number of open ports
- Overall scan performance

📸 **Screenshot:** Default timing scan

---

# Task 2: Aggressive Timing Scan

## Command

```bash
nmap -T4 scanme.nmap.org
```

### Observe

- Compare scan duration with the default scan.
- Check if the same ports are detected.

📸 **Screenshot:** T4 scan

---

# Task 3: Insane Timing Scan

## Command

```bash
nmap -T5 scanme.nmap.org
```

### Observe

- Scan speed
- Accuracy
- Any missing ports

📸 **Screenshot:** T5 scan

---

# Task 4: Scan with Delay

## Command

```bash
nmap --scan-delay 1s scanme.nmap.org
```

### Observe

- Increase in scan duration
- Network behavior

📸 **Screenshot:** Scan delay

---

# Task 5: Limit Packet Retries

## Command

```bash
nmap --max-retries 2 scanme.nmap.org
```

### Observe

- Reduction in scan time
- Any differences in detected ports

📸 **Screenshot:** Maximum retries

---

# Task 6: High Packet Rate

## Command

```bash
nmap --min-rate 1000 scanme.nmap.org
```

### Observe

- Faster packet transmission
- Overall scan completion time

📸 **Screenshot:** High packet rate

---

# Task 7: Optimized Scan

## Command

```bash
nmap -T4 --min-rate 1000 --max-retries 2 scanme.nmap.org
```

### Observe

- Scan duration
- Accuracy
- Compare with previous scans

📸 **Screenshot:** Optimized scan

# Comparison Table

| Scan | Speed | Accuracy | Network Load | Recommended Use |
|------|-------|----------|--------------|-----------------|
| Default (`-T3`) | Medium | High | Medium | General scanning |
| `-T4` | Fast | High | High | Internal networks |
| `-T5` | Very Fast | Medium | Very High | Lab environments |
| `--scan-delay` | Slow | High | Low | Stealth scanning |
| `--max-retries` | Fast | Medium | Medium | Faster assessments |
| `--min-rate` | Fast | High | High | Large network scans |

# Analysis Questions

1. Which timing template completed the scan fastest?
2. Which timing template produced the most reliable results?
3. Did `-T5` miss any information compared to `-T3`?
4. How did `--scan-delay` affect scan duration?
5. When should `--max-retries` be reduced?
6. Why should `-T5` generally be avoided on production networks?
7. Which timing option would you choose for a large internal network?

# Challenge

Run the following command:

```bash
sudo nmap -T4 -A -sV -O -oA optimized_scan scanme.nmap.org
```

After the scan:

- Compare the generated reports.
- Record the total scan time.
- Note all detected services and OS information.
- Explain why `-T4` was a good choice for this scan.

---

# Best Practices

- Use `-T3` when unsure.
- Use `-T4` on trusted, fast networks.
- Avoid `-T5` on production systems.
- Tune packet rates carefully.
- Validate critical findings with follow-up scans.
