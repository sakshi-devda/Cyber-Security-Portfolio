# Scan Techniques (Practical)

## 🎯 Objective

In this practical, you will perform different Nmap scan techniques and compare their results.

---

# Lab Environment

- Operating System: Kali Linux
- Tool: Nmap
- Target: scanme.nmap.org or Your Lab Machine

---

# Practical 1: TCP Connect Scan

## Command

```bash
nmap -sT scanme.nmap.org
```

### Purpose

Performs a full TCP three-way handshake to identify open ports.

### Expected Output

- Open ports
- Running services

📸 **Screenshot:** TCP Connect Scan

---

# Practical 2: SYN Scan

## Command

```bash
sudo nmap -sS scanme.nmap.org
```

### Purpose

Performs a half-open (stealth) scan without completing the TCP handshake.

### Expected Output

- Open ports
- Faster scan results

📸 **Screenshot:** SYN Scan

---

# Practical 3: UDP Scan

## Command

```bash
sudo nmap -sU scanme.nmap.org
```

### Purpose

Scans UDP ports to identify UDP services.

### Expected Output

- Open UDP ports
- UDP services (if detected)

📸 **Screenshot:** UDP Scan

# Practical 4: FIN Scan

## Command

```bash
sudo nmap -sF scanme.nmap.org
```

### Purpose

# Practical 7: ACK Scan

## Command

```bash
sudo nmap -sA scanme.nmap.org
```

# Practical 8: Idle Scan

## Command

```bash
sudo nmap -sI <zombie-host> scanme.nmap.org
```

### Example

```bash
sudo nmap -sI 192.168.1.20 scanme.nmap.org
```

### Purpose

Performs a stealth scan using an idle (zombie) host.

> **Note:** This scan requires a properly configured zombie host and may not work in every lab environment.

📸 **Screenshot:** Idle Scan (if available)

# Practical Summary

| Scan Type | Option | Purpose |
|-----------|--------|---------|
| TCP Connect | `-sT` | Full TCP handshake |
| SYN | `-sS` | Half-open scan |
| UDP | `-sU` | Scan UDP ports |
| FIN | `-sF` | FIN flag scan |
| NULL | `-sN` | No TCP flags |
| Xmas | `-sX` | FIN + PSH + URG flags |
| ACK | `-sA` | Firewall analysis |
| Idle | `-sI` | Stealth scan using a zombie host |

---

# Key Observations

- TCP Connect Scan completes the full handshake.
- SYN Scan is faster and more stealthy.
- UDP Scan usually takes longer.
- FIN, NULL, and Xmas Scans may behave differently depending on the target operating system and firewall.
- ACK Scan helps identify firewall filtering.
- Idle Scan provides a high level of stealth but requires additional setup.

---

# Best Practices

- Perform scans only on authorized systems.
- Compare scan results to understand different techniques.
- Save screenshots for documentation.
- Analyze unexpected port states carefully.
