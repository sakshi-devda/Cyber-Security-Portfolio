# Nmap Installation and First Scan (Practical)

## 🎯 Objective

In this practical, you will:

- Verify that Nmap is installed.
- Check the installed version.
- View the help menu.
- Perform your first network scan.
- Understand the basic scan output.

---

# Lab Environment

- Operating System: Kali Linux
- Tool: Nmap
- User: kali

---

# Step 1: Verify Nmap Installation

Run the following command:

```bash
nmap --version
```

### Expected Output

The output should display the installed version of Nmap along with additional information such as the libraries it uses.

📸 **Screenshot:** `nmap --version`

---

# Step 2: View the Help Menu

Run:

```bash
nmap --help
```

### Purpose

Displays all available commands, options, and scan types supported by Nmap.

📸 **Screenshot:** `nmap --help`

---

# Step 3: Perform Your First Scan

Run:

```bash
nmap scanme.nmap.org
```

### Purpose

Scans the publicly available test server provided by the Nmap project.

This is a safe target designed for learning and practice.

📸 **Screenshot:** First scan output

---

# Step 4: Understand the Output

A typical Nmap scan includes:

## Target Host

Shows the system being scanned.

---

## Host Status

Example:

```
Host is up
```

This means the target is reachable.

---

## Open Ports

Example:

```
22/tcp open ssh
80/tcp open http
```

This indicates:

- Port 22 is running SSH.
- Port 80 is running HTTP.

---

## Service Name

Nmap identifies common services associated with open ports.

Example:

| Port | Service |
|------|---------|
| 22 | SSH |
| 80 | HTTP |
| 443 | HTTPS |

---

## Scan Summary

Example:

```
Nmap done:
1 IP address scanned
```

This shows that the scan has completed successfully.

📸 **Screenshot:** Final scan summary

# Common Errors

## Command Not Found

Cause:

Nmap is not installed or is not in the system PATH.

Solution:

Install Nmap using the package manager.

---

## Host Seems Down

Cause:

The target is unreachable or blocking ping requests.

Solution:

Verify the target address and network connectivity.

---

## Permission Denied

Cause:

Some scan types require administrator (root) privileges.

Solution:

Run the command with:

```bash
sudo
```

when required.

---

# Best Practices

- Scan only systems you own or have permission to test.
- Avoid scanning unknown or unauthorized systems.
- Save scan results for documentation.
- Start with basic scans before using advanced options.
