# OS Detection (Practical)

## 🎯 Objective

In this practical, you will learn how to detect the operating system of a target using Nmap and analyze the results.

---

# Lab Environment

- OS: Kali Linux
- Tool: Nmap
- Target: scanme.nmap.org or lab machine

---

# Practical 1: Basic OS Detection

## Command

```bash
sudo nmap -O scanme.nmap.org
```

### Purpose

Detects the operating system of the target system using TCP/IP stack fingerprinting.

### Expected Output

- OS type (Linux / Windows / etc.)
- Network distance
- Device type (if detected)

📸 **Screenshot:** Basic OS detection

---

# Practical 2: OS Detection with Service Scan

## Command

```bash
sudo nmap -O -sV scanme.nmap.org
```

### Purpose

Combines OS detection with service version detection for better accuracy.

### Expected Output

- Open ports
- Running services
- OS details

📸 **Screenshot:** OS + Service detection

---

# Practical 3: Aggressive OS Detection

## Command

```bash
sudo nmap -A scanme.nmap.org
```

### Purpose

Performs aggressive scanning which includes:
- OS detection
- Version detection
- Script scanning
- Traceroute

📸 **Screenshot:** Aggressive scan output

---

# Practical 4: OS Detection on Local Network

## Command

```bash
sudo nmap -O 192.168.1.0/24
```

### Purpose

Attempts to identify operating systems of all devices in a local network.

📸 **Screenshot:** LAN OS detection

---

# Practical 5: Verbose OS Detection

## Command

```bash
sudo nmap -O -v scanme.nmap.org
```

### Purpose

Provides detailed output during OS detection.

📸 **Screenshot:** Verbose OS scan

---

# Output Interpretation

Example output:

```
OS details: Linux 5.x
Network Distance: 0 hops
```

OR

```
OS details: Windows 10 / Server 2019
```

---

## 🧠 Analysis

| Field | Meaning |
|------|--------|
| OS details | Detected operating system |
| Network distance | Number of hops to target |
| Device type | Server, router, etc. |

---

# 🔎 Key Observations

- `-O` requires root privileges.
- Accuracy improves when combined with `-sV`.
- Aggressive scan (`-A`) gives the most complete results.
- LAN scans are usually more accurate than remote scans.

---

# 🛡️ Security Perspective

OS detection helps security professionals:

- Identify system types in a network.
- Detect outdated operating systems.
- Plan vulnerability assessments.
- Build asset inventory.

---

# 💼 Real-World Example

A security team scans the internal network and finds:

```
192.168.1.15 → Windows 7
```

Since Windows 7 is no longer supported, the system is flagged for immediate upgrade due to security risks.

---

# ⭐ Best Practices

- Always run with sudo/root privileges.
- Combine with `-sV` for better accuracy.
- Avoid scanning unknown external systems.
- Use `-A` only in controlled environments.
- Verify results with multiple scans.
