# HTTP vs HTTPS

## 📌 What are HTTP and HTTPS?

HTTP (HyperText Transfer Protocol) and HTTPS (HyperText Transfer Protocol Secure) are application-layer protocols used to transfer data between a web browser and a web server.

The key difference is that **HTTPS encrypts communication**, while **HTTP does not**.

---

## 🎯 Why are HTTP and HTTPS Important?

They help users:

- Access websites.
- Transfer web pages and data.
- Submit forms.
- Download files.
- Communicate securely over the Internet.

---

# 🌐 What is HTTP?

HTTP is a protocol used to transfer web pages between a client (browser) and a web server.

### Features

- Fast communication
- No encryption
- Data is transmitted in plain text
- Less secure

### Default Port

```text
80
```

---

# 🔒 What is HTTPS?

HTTPS is the secure version of HTTP.

It uses **SSL/TLS (Secure Sockets Layer / Transport Layer Security)** to encrypt data during transmission.

### Features

- Encrypted communication
- Protects sensitive information
- Ensures data confidentiality and integrity
- Authenticates the website using a digital certificate

### Default Port

```text
443
```

---

# 📊 HTTP vs HTTPS

| HTTP | HTTPS |
|------|--------|
| Full Form: HyperText Transfer Protocol | HyperText Transfer Protocol Secure |
| Default Port: 80 | Default Port: 443 |
| Data is not encrypted | Data is encrypted using SSL/TLS |
| Less secure | More secure |
| Vulnerable to interception | Protects against eavesdropping and tampering |
| No digital certificate required | Requires an SSL/TLS certificate |

---

# 🔐 What is SSL/TLS?

SSL (Secure Sockets Layer) and its successor, TLS (Transport Layer Security), are cryptographic protocols that secure communication between a client and a server.

They provide:

- Encryption
- Authentication
- Data integrity

Today, **TLS** is the modern standard, although people often still use the term "SSL certificate."

---

# 🌍 How HTTPS Works

1. A user enters a website URL.
2. The browser connects to the web server.
3. The server presents its SSL/TLS certificate.
4. A secure encrypted connection is established.
5. Data is exchanged securely.

---

## 🛡️ HTTP and HTTPS in Cybersecurity

Cybersecurity professionals prefer HTTPS because it:

- Protects login credentials.
- Secures financial transactions.
- Prevents attackers from reading intercepted traffic.
- Reduces the risk of Man-in-the-Middle (MITM) attacks.

Using plain HTTP for sensitive information can expose usernames, passwords, and other confidential data.

---

## 💼 Real-World Example

When you log in to your online banking account:

- Your browser uses **HTTPS**.
- Login credentials are encrypted.
- Attackers cannot easily read the transmitted data, even if they intercept the traffic.

---

## ⭐ Best Practices

- Always use HTTPS for websites handling sensitive data.
- Verify the website's digital certificate.
- Avoid entering passwords on HTTP websites.
- Keep SSL/TLS certificates updated.
- Disable outdated SSL versions and use modern TLS versions.

## 🎯 Key Takeaway

HTTP is used to transfer web data, but it does not provide encryption. HTTPS secures communication using TLS, protecting data from interception and tampering. It is the standard protocol for secure web communication and is essential for modern cybersecurity.
