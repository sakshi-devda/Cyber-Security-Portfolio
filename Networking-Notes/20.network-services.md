# Network Services

## 📌 What are Network Services?

Network services are applications or protocols that enable communication, resource sharing, and management across a network. They allow devices to exchange data and access resources such as websites, email, files, and remote systems.

These services form the backbone of modern computer networks.

---

## 🎯 Why are Network Services Important?

Network services help to:

- Enable communication between devices.
- Provide Internet connectivity.
- Support file sharing.
- Deliver email services.
- Allow remote administration.
- Synchronize time across devices.
- Manage network infrastructure.

---

# 🌐 Common Network Services

## 1. DNS (Domain Name System)

### Purpose

Translates domain names into IP addresses.

### Default Port

- UDP 53
- TCP 53

### Example

```
www.google.com → 142.250.x.x
```

---

## 2. DHCP (Dynamic Host Configuration Protocol)

### Purpose

Automatically assigns IP addresses and network settings to devices.

### Default Ports

- UDP 67 (Server)
- UDP 68 (Client)

---

## 3. HTTP (HyperText Transfer Protocol)

### Purpose

Transfers web pages between browsers and web servers.

### Default Port

- TCP 80

---

## 4. HTTPS (HyperText Transfer Protocol Secure)

### Purpose

Securely transfers web pages using TLS encryption.

### Default Port

- TCP 443

---

## 5. FTP (File Transfer Protocol)

### Purpose

Transfers files between systems.

### Default Ports

- TCP 20 (Data)
- TCP 21 (Control)

---

## 6. SSH (Secure Shell)

### Purpose

Provides secure remote access to systems.

### Default Port

- TCP 22

---

## 7. Telnet

### Purpose

Provides remote command-line access without encryption.

### Default Port

- TCP 23

### Security Note

Telnet is considered insecure and has largely been replaced by SSH.

---

## 8. SMTP (Simple Mail Transfer Protocol)

### Purpose

Sends email between mail servers.

### Default Port

- TCP 25

---

## 9. POP3 (Post Office Protocol Version 3)

### Purpose

Downloads email from a mail server to a client.

### Default Port

- TCP 110

---

## 10. IMAP (Internet Message Access Protocol)

### Purpose

Allows users to access and manage email directly on the mail server.

### Default Port

- TCP 143

---

## 11. SNMP (Simple Network Management Protocol)

### Purpose

Monitors and manages network devices such as routers, switches, and printers.

### Default Ports

- UDP 161
- UDP 162 (Traps)

---

## 12. NTP (Network Time Protocol)

### Purpose

Synchronizes time across devices on a network.

### Default Port

- UDP 123

---

## 🛡️ Network Services in Cybersecurity

Cybersecurity professionals monitor network services to:

- Detect unauthorized services.
- Identify vulnerable applications.
- Verify secure configurations.
- Investigate suspicious activity.
- Reduce the attack surface.

For example, an exposed Telnet service should be disabled and replaced with SSH.

---

## 💼 Real-World Example

A security analyst performs a network scan and discovers:

- Port 22 (SSH) — Open
- Port 80 (HTTP) — Open
- Port 443 (HTTPS) — Open
- Port 23 (Telnet) — Open

The analyst recommends disabling Telnet because it transmits data without encryption and replacing it with SSH.

---

## ⭐ Best Practices

- Disable unused network services.
- Replace insecure protocols with secure alternatives.
- Keep services updated.
- Monitor open ports regularly.
- Restrict access using firewalls.
- Follow the principle of least privilege.
