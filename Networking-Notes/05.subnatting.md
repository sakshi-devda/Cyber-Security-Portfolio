# Subnetting

## 📌 What is Subnetting?

Subnetting is the process of dividing a large network into smaller, more manageable networks called **subnets**.

It helps improve network performance, security, and efficient use of IP addresses.

---

## 🎯 Why is Subnetting Important?

Subnetting helps to:

- Reduce network congestion.
- Improve network performance.
- Increase security by separating networks.
- Use IP addresses more efficiently.
- Simplify network management.

---

## 🧠 Key Terms

### Network

A group of connected devices that communicate with each other.

---

### Subnet

A smaller section of a larger network.

Example:

A company may divide its network into:

- HR Department
- Finance Department
- IT Department

Each department can have its own subnet.

---

### Subnet Mask

A subnet mask identifies which part of an IP address represents the **network** and which part represents the **host**.

Example:

```text
IP Address : 192.168.1.25
Subnet Mask: 255.255.255.0
```

Here:

- Network Portion → 192.168.1
- Host Portion → 25

---

---

## 📖 What is CIDR?

CIDR (Classless Inter-Domain Routing) is a method of representing IP addresses and subnet masks.

Example:

```text
192.168.1.0/24
```

Here:

- `192.168.1.0` → Network Address
- `/24` → First 24 bits represent the network.

---

## 📡 Benefits of Subnetting

- Better network organization.
- Reduced broadcast traffic.
- Improved security.
- Efficient IP address allocation.
- Easier troubleshooting.

---

## 🛡️ Subnetting in Cybersecurity

Subnetting is important because it:

- Separates sensitive systems from public networks.
- Limits the spread of cyber attacks.
- Improves access control.
- Helps monitor network traffic more effectively.

For example, a company may place employee systems, servers, and guest Wi-Fi on separate subnets to improve security.

---

## 💼 Real-World Example

A company has 300 employees.

Instead of placing all devices on one network, the administrator creates separate subnets for:

- Human Resources
- Finance
- IT
- Guest Wi-Fi

This improves performance, simplifies management, and enhances security.

---

## ⭐ Best Practices

- Plan IP address allocation before creating subnets.
- Separate critical systems from user devices.
- Use VLANs with subnetting for better segmentation.
- Document subnet ranges clearly.
- Regularly review network design as it grows.
