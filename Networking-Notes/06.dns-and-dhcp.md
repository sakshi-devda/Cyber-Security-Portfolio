# DNS and DHCP

## 📌 What are DNS and DHCP?

DNS (Domain Name System) and DHCP (Dynamic Host Configuration Protocol) are two essential networking services.

- **DNS** translates domain names into IP addresses.
- **DHCP** automatically assigns IP addresses and other network settings to devices.

Together, they make network communication simple and efficient.

---

## 🎯 Why are DNS and DHCP Important?

- DNS makes websites easy to access using names instead of IP addresses.
- DHCP automatically configures devices when they join a network.
- Both reduce manual configuration and improve network management.

---

# 🌐 DNS (Domain Name System)

## What is DNS?

DNS is often called the **Phonebook of the Internet**.

Instead of remembering an IP address like:

```text
142.250.183.78
```

You simply type:

```text
www.google.com
```

DNS converts the domain name into its corresponding IP address.

---

## How DNS Works

1. You enter a website name in your browser.
2. A DNS server looks up the corresponding IP address.
3. The browser receives the IP address.
4. The browser connects to the web server.

---

## Common DNS Record Types

| Record | Purpose |
|---------|---------|
| A | Maps a domain to an IPv4 address |
| AAAA | Maps a domain to an IPv6 address |
| CNAME | Creates an alias for another domain |
| MX | Specifies the mail server |
| TXT | Stores text information (e.g., verification records) |

---

# 📡 DHCP (Dynamic Host Configuration Protocol)

## What is DHCP?

DHCP is a protocol that automatically assigns network settings to devices.

These settings include:

- IP Address
- Subnet Mask
- Default Gateway
- DNS Server

---

## How DHCP Works

When a device connects to a network:

1. The device requests an IP address.
2. The DHCP server offers an available IP address.
3. The device accepts the offer.
4. The DHCP server confirms the assignment.

This process is known as **DORA**:

- Discover
- Offer
- Request
- Acknowledge

---

## Static IP vs DHCP

| Static IP | DHCP |
|------------|------|
| Manually configured | Automatically assigned |
| Fixed address | Address may change |
| Common for servers | Common for user devices |

---

## 🛡️ DNS and DHCP in Cybersecurity

Cybersecurity professionals use DNS and DHCP logs to:

- Investigate suspicious domains.
- Detect malware communication.
- Identify rogue DHCP servers.
- Analyze network activity.
- Track infected devices.

---

## 💼 Real-World Example

When you connect your laptop to your home Wi-Fi:

- The **DHCP server** assigns your device an IP address.
- When you open **www.google.com**, the **DNS server** translates the domain name into an IP address so your browser can connect to Google's server.

---

## ⭐ Best Practices

- Use trusted DNS servers.
- Monitor DNS queries for suspicious domains.
- Secure DHCP servers against unauthorized access.
- Reserve static IP addresses for critical systems.
- Regularly review DNS and DHCP logs.
