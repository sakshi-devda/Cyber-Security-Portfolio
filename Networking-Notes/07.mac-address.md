# MAC Address

## 📌 What is a MAC Address?

A MAC (Media Access Control) Address is a unique physical address assigned to a Network Interface Card (NIC) by the manufacturer. It is used to identify a device on a Local Area Network (LAN).

Unlike an IP address, which can change, a MAC address is usually permanent.

---

## 🎯 Why is a MAC Address Important?

A MAC address is important because it:

- Uniquely identifies a device on a local network.
- Enables communication within a LAN.
- Helps switches forward data correctly.
- Is used for network security and device identification.

---

## 🧠 MAC Address Format

A MAC address is a **48-bit (6-byte)** hexadecimal value.

Example:

```text
00:1A:2B:3C:4D:5E
```

It can also be written as:

```text
00-1A-2B-3C-4D-5E
```

Each pair represents one byte in hexadecimal format.

---

## 🌐 Structure of a MAC Address

A MAC address consists of two parts:

### 1. OUI (Organizationally Unique Identifier)

- First 24 bits
- Identifies the manufacturer

### 2. Device Identifier

- Last 24 bits
- Unique identifier assigned by the manufacturer

---

## 📡 MAC Address vs IP Address

| MAC Address | IP Address |
|-------------|------------|
| Physical address | Logical address |
| Assigned by manufacturer | Assigned manually or by DHCP |
| Usually permanent | Can change |
| Used in Local Area Networks | Used across different networks |
| Operates at the Data Link Layer | Operates at the Network Layer |

---

## 🔄 How MAC Addresses Work

When two devices communicate on the same local network:

1. The sender identifies the destination MAC address.
2. The switch forwards the frame using the MAC address.
3. The destination device receives the data.

For communication outside the local network, routers use IP addresses.

---

## 🛡️ MAC Address in Cybersecurity

MAC addresses help security professionals:

- Identify devices on a network.
- Detect unauthorized devices.
- Investigate network incidents.
- Apply MAC filtering for access control.

However, attackers can perform **MAC spoofing**, where they change their device's MAC address to impersonate another device.

---

## 💼 Real-World Example

A company allows only registered office laptops to connect to its Wi-Fi.

The wireless controller checks each device's MAC address before granting access.

If an unknown MAC address attempts to connect, the connection is blocked.

---

## ⭐ Best Practices

- Monitor new or unknown MAC addresses on the network.
- Use MAC filtering as an additional security measure (not the only one).
- Regularly review connected devices.
- Combine MAC-based controls with strong authentication.
