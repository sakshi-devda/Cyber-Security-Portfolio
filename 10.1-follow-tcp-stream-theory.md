# Follow TCP Stream

## 🎯 Objective

Understand the Follow TCP Stream feature in Wireshark, how it works, and how it helps reconstruct complete TCP conversations for network analysis and cybersecurity investigations.

---

# What is Follow TCP Stream?

Follow TCP Stream is one of the most useful features in Wireshark that reconstructs the complete communication between a client and a server over a TCP connection.

Instead of viewing individual packets separately, Wireshark combines all packets belonging to the same TCP session into a single readable conversation.

This makes it much easier to understand what data was exchanged during the communication.

---

# Why is Follow TCP Stream Important?

During network communication, application data is divided into multiple TCP packets.

Analyzing each packet individually can be time-consuming and confusing.

Follow TCP Stream reconstructs the entire conversation, allowing analysts to quickly understand the communication without manually examining every packet.

---

# How Follow TCP Stream Works

The process works as follows:

1. A TCP connection is established between a client and a server.
2. Data is exchanged through multiple TCP packets.
3. Wireshark identifies packets belonging to the same TCP session.
4. Follow TCP Stream combines all these packets into one continuous conversation.

---

# TCP Stream Communication Flow

```
Client
   │
   ▼
TCP Connection Established
   │
   ▼
Request Sent
   │
   ▼
Server Response
   │
   ▼
TCP Connection Closed
```

---

# Information Displayed

The Follow TCP Stream window may display:

- HTTP Requests
- HTTP Responses
- URLs
- Cookies
- HTML Content
- Plain Text Data
- Login Credentials (if transmitted without encryption)
- Commands exchanged between systems

---

# Supported Protocols

Follow TCP Stream works with all TCP-based protocols, including:

- HTTP
- HTTPS (Encrypted Data)
- FTP
- SMTP
- POP3
- IMAP
- SSH
- Telnet

---

# How to Open Follow TCP Stream

In Wireshark:

1. Select any TCP packet.
2. Right-click the packet.
3. Click **Follow**.
4. Select **TCP Stream**.

A new window will display the complete conversation.

---

# Display Options

The Follow TCP Stream window provides several display formats, such as:

- ASCII
- EBCDIC
- Hex Dump
- C Arrays
- Raw Data
- UTF-8

These options help analysts view data in different formats depending on the protocol and investigation.

---

# TCP Stream Number

Each TCP conversation is assigned a unique **Stream Number**.

This allows analysts to switch between multiple TCP sessions captured in the same packet capture file.

---

# Limitations

Follow TCP Stream has some limitations:

- It cannot decrypt encrypted HTTPS traffic.
- It only reconstructs packets that were successfully captured.
- Missing packets may result in incomplete conversations.

---

# Follow TCP Stream in Cyber Security

Security professionals use this feature to:

- Analyze suspicious web traffic.
- Investigate malware communication.
- Detect sensitive information sent in plain text.
- Examine client-server conversations.
- Support digital forensic investigations.
- Troubleshoot application communication issues.

---

# Advantages

- Reconstructs complete TCP conversations.
- Saves analysis time.
- Makes packet analysis easier.
- Displays application-layer data in a readable format.
- Supports multiple TCP-based protocols.

---

# Limitations

- Cannot decrypt encrypted traffic.
- Depends on complete packet capture.
- Does not work with UDP traffic.

---

# Best Practices

- Apply Display Filters before following a stream.
- Verify the correct TCP session.
- Save important packet captures.
- Handle sensitive information securely during analysis.
