## 2. Objective

The objective of this room is to understand the purpose of an operating system (OS), its core functions, and how it manages computer hardware and software resources.

---

## 3. Introduction

An Operating System (OS) is system software that acts as an interface between the user and the computer hardware. It manages system resources, executes applications, controls hardware devices, and provides a secure environment for users and programs.

Without an operating system, a computer cannot efficiently run applications or manage hardware components.

---

## 4. Key Concepts

### What is an Operating System?

An Operating System is the software responsible for controlling and coordinating the activities of a computer system. It provides the foundation on which all other software runs.

Examples:

- Microsoft Windows
- Linux
- macOS
- Android
- iOS

---

### Process Management

The operating system manages running programs (processes), allocates CPU time, and ensures multiple applications can run efficiently.

---

### Memory Management

The OS manages RAM by allocating memory to running programs and freeing it when applications are closed.

---

### File Management

The operating system organizes and manages files and directories stored on storage devices.

Common operations include:

- Create files
- Delete files
- Copy files
- Move files
- Rename files

---

### Device Management

The OS communicates with hardware devices through device drivers.

Examples:

- Keyboard
- Mouse
- Printer
- Network Adapter
- Monitor

---

### User Management

Operating systems support multiple user accounts and control access through authentication and permissions.

---

### Security Features

Modern operating systems include built-in security features such as:

- User Authentication
- File Permissions
- Firewall
- Encryption
- Automatic Updates
- Access Control

---

## 5. Types of Operating Systems

Some common operating systems include:

- Windows
- Linux
- macOS
- Android
- iOS
- Unix

Each operating system is designed for specific environments and use cases.
