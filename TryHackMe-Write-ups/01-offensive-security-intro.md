## 2. Objective

The objective of this room is to introduce the fundamentals of Offensive Security and explain how ethical hackers identify and exploit vulnerabilities to help organizations improve their security.

---

## 3. What is Offensive Security?

Offensive Security is the practice of legally testing computer systems, networks, and applications to identify security weaknesses before malicious attackers can exploit them.

Security professionals performing these activities are known as:

- Ethical Hackers
- Penetration Testers
- Red Team Members

Their goal is to improve an organization's security by discovering vulnerabilities in a controlled and authorized manner.

---

## 4. Key Concepts

### Ethical Hacking

Ethical hacking is the process of legally testing systems for security vulnerabilities with the owner's permission.

### Penetration Testing

Penetration testing is a simulated cyber attack performed to identify and validate security weaknesses.

### Vulnerability

A vulnerability is a weakness in software, hardware, or network configurations that can be exploited by attackers.

### Exploit

An exploit is a method or piece of code used to take advantage of a vulnerability.

### Capture The Flag (CTF)

Capture The Flag (CTF) challenges are hands-on cybersecurity exercises where participants solve security-related tasks to find hidden flags.

## 8. Key Takeaways

- Offensive Security focuses on finding vulnerabilities before attackers.
- Ethical hacking is performed only with proper authorization.
- Penetration testing helps improve an organization's security posture.
- Practical labs like CTFs develop problem-solving and technical skills.
