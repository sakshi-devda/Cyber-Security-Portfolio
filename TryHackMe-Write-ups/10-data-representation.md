## 2. Objective

The objective of this room is to understand how computers represent and process data using binary, decimal, hexadecimal, and other numerical systems.

---

## 3. Introduction

Computers do not understand human languages or decimal numbers directly. Instead, they represent and process all information using binary digits (bits). Every character, number, image, audio file, and program is ultimately stored and processed as binary data.

Understanding data representation is fundamental in cybersecurity because analysts frequently work with binary, hexadecimal values, memory dumps, packet captures, and encoded data.

---

## 4. Key Concepts

### Binary Number System

Binary is the base-2 number system used by computers.

It contains only two digits:

- 0
- 1

Example:

- Decimal 5 = Binary 101
- Decimal 10 = Binary 1010

---

### Decimal Number System

The decimal system is the standard number system used in everyday life.

It uses ten digits:

- 0–9

Example:

- 25
- 100
- 256

---

### Hexadecimal Number System

Hexadecimal (Hex) is a base-16 numbering system commonly used in computing because it represents binary values in a compact form.

It uses:

- 0–9
- A–F

Example:

- Decimal 255 = Hex FF
- Binary 11111111 = Hex FF

---

### Bits and Bytes

A **Bit** is the smallest unit of data and can have a value of either 0 or 1.

A **Byte** consists of 8 bits.

Common data units:

- 1 Byte = 8 Bits
- 1 KB = 1024 Bytes
- 1 MB = 1024 KB
- 1 GB = 1024 MB
- 1 TB = 1024 GB

---

### Character Encoding

Computers store letters, numbers, and symbols using standardized character encoding systems.

Common standards include:

- ASCII
- Unicode
- UTF-8

These standards ensure data can be correctly interpreted across different systems.
